## 一、设计思想

### 1.1 中间件系统概述

DeerFlow 的中间件系统是基于 LangChain `AgentMiddleware` 的请求处理管道，在 Agent 执行前后插入自定义逻辑。核心功能包括：

- **生命周期钩子**: `before_agent`, `after_agent`, `before_model`, `after_model`, `wrap_tool_call`
- **状态修改**: 通过返回字典更新 Agent 状态
- **执行流控制**: 通过 `Command(goto=END)` 中断执行
- **并发安全**: 支持同步和异步方法

### 1.3 核心设计决策

**为什么需要中间件链？**

1. **关注点分离**: 每个中间件专注于单一职责
2. **可组合性**: 可以灵活启用/禁用特定中间件
3. **可测试性**: 每个中间件可以独立测试

**为什么使用装饰器模式？**

1. **透明性**: 中间件对 Agent 透明，无需修改 Agent 代码
2. **灵活性**: 可以在执行前后插入逻辑
3. **链式调用**: 多个中间件可以串联执行

**为什么有同步和异步方法？**

1. **性能优化**: 异步方法避免阻塞事件循环
2. **兼容性**: 同步方法用于不支持异步的场景
3. **框架支持**: LangGraph 同时支持两种调用方式

**为什么需要 wrap_tool_call？**

1. **工具级拦截**: 可以在工具执行前后插入逻辑
2. **执行流控制**: 可以中断工具执行 (如 ClarificationMiddleware)
3. **参数修改**: 可以修改工具调用参数

**源码位置**: `packages/harness/deerflow/agents/middlewares/thread_data_middleware.py:21-100`

**职责**: 为每个线程创建隔离的数据目录

```python
class ThreadDataMiddleware(AgentMiddleware[ThreadDataMiddlewareState]):
    """Create thread data directories for each thread execution.

    Creates:
    - {base_dir}/threads/{thread_id}/user-data/workspace
    - {base_dir}/threads/{thread_id}/user-data/uploads
    - {base_dir}/threads/{thread_id}/user-data/outputs
    """

    def __init__(self, base_dir: str | None = None, lazy_init: bool = True):
        super().__init__()
        self._paths = Paths(base_dir) if base_dir else get_paths()
        self._lazy_init = lazy_init

    @override
    def before_agent(self, state: ThreadDataMiddlewareState, runtime: Runtime) -> dict | None:
        context = runtime.context or {}
        thread_id = context.get("thread_id")
        if thread_id is None:
            config = get_config()
            thread_id = config.get("configurable", {}).get("thread_id")

        if thread_id is None:
            raise ValueError("Thread ID is required")

        if self._lazy_init:
            paths = self._get_thread_paths(thread_id)
        else:
            paths = self._create_thread_directories(thread_id)

        return {"thread_data": {**paths}}
```

**设计要点**:
1. **懒加载**: `lazy_init=True` 时仅计算路径，不创建目录
2. **路径隔离**: 每个线程有独立的 workspace/uploads/outputs
3. **虚拟路径映射**: 为后续路径替换提供基础

### 3.3 LoopDetectionMiddleware - 循环检测

**源码位置**: `packages/harness/deerflow/agents/middlewares/loop_detection_middleware.py:69-228`

**职责**: 检测并中断重复的工具调用循环

```python
class LoopDetectionMiddleware(AgentMiddleware[AgentState]):
    """Detects and breaks repetitive tool call loops."""

    def __init__(
        self,
        warn_threshold: int = 3,  # 警告阈值
        hard_limit: int = 5,      # 强制停止阈值
        window_size: int = 20,    # 滑动窗口大小
        max_tracked_threads: int = 100,  # 最大跟踪线程数
    ):
        super().__init__()
        self.warn_threshold = warn_threshold
        self.hard_limit = hard_limit
        self.window_size = window_size
        self._lock = threading.Lock()
        self._history: OrderedDict[str, list[str]] = OrderedDict()  # LRU
        self._warned: dict[str, set[str]] = defaultdict(set)

    def _track_and_check(self, state: AgentState, runtime: Runtime) -> tuple[str | None, bool]:
        """Track tool calls and check for loops."""
        tool_calls = getattr(last_msg, "tool_calls", None)
        if not tool_calls:
            return None, False

        thread_id = self._get_thread_id(runtime)
        call_hash = _hash_tool_calls(tool_calls)  # 工具调用的确定性哈希

        with self._lock:
            # LRU 更新
            if thread_id in self._history:
                self._history.move_to_end(thread_id)
            else:
                self._history[thread_id] = []
                self._evict_if_needed()

            history = self._history[thread_id]
            history.append(call_hash)
            if len(history) > self.window_size:
                history[:] = history[-self.window_size:]

            count = history.count(call_hash)

            if count >= self.hard_limit:
                return _HARD_STOP_MSG, True
            if count >= self.warn_threshold:
                if call_hash not in self._warned[thread_id]:
                    self._warned[thread_id].add(call_hash)
                    return _WARNING_MSG, False

        return None, False

    def _apply(self, state: AgentState, runtime: Runtime) -> dict | None:
        warning, hard_stop = self._track_and_check(state, runtime)

        if hard_stop:
            # 移除 tool_calls 强制文本输出
            stripped_msg = last_msg.model_copy(update={"tool_calls": []})
            return {"messages": [stripped_msg]}

        if warning:
            # 注入警告消息
            return {"messages": [HumanMessage(content=warning)]}

        return None
```

**设计要点**:
1. **哈希检测**: 对工具调用名称+参数进行哈希
2. **滑动窗口**: 只追踪最近 N 次调用
3. **两级阈值**: 先警告后强制停止
4. **LRU 缓存**: 自动清理旧线程的追踪数据

### 3.4 SubagentLimitMiddleware - 子代理数量限制

**源码位置**: `packages/harness/deerflow/agents/middlewares/subagent_limit_middleware.py:24-76`

**职责**: 限制单次响应中的并发子代理调用数量

```python
class SubagentLimitMiddleware(AgentMiddleware[AgentState]):
    """Truncates excess 'task' tool calls from a single model response."""

    def __init__(self, max_concurrent: int = MAX_CONCURRENT_SUBAGENTS):
        super().__init__()
        self.max_concurrent = _clamp_subagent_limit(max_concurrent)  # 限制在 [2, 4]

    def _truncate_task_calls(self, state: AgentState) -> dict | None:
        messages = state.get("messages", [])
        if not messages:
            return None

        last_msg = messages[-1]
        if getattr(last_msg, "type", None) != "ai":
            return None

        tool_calls = getattr(last_msg, "tool_calls", None)
        if not tool_calls:
            return None

        # 找出所有 task 工具调用的索引
        task_indices = [i for i, tc in enumerate(tool_calls) if tc.get("name") == "task"]
        if len(task_indices) <= self.max_concurrent:
            return None

        # 丢弃超出限制的 task 调用
        indices_to_drop = set(task_indices[self.max_concurrent :])
        truncated_tool_calls = [tc for i, tc in enumerate(tool_calls) if i not in indices_to_drop]

        # 替换 AIMessage (相同 id 触发替换)
        updated_msg = last_msg.model_copy(update={"tool_calls": truncated_tool_calls})
        return {"messages": [updated_msg]}
```

**设计要点**:
1. **截断策略**: 保留前 N 个，丢弃多余的
2. **范围限制**: max_concurrent 限制在 [2, 4]
3. **消息替换**: 使用相同 id 触发 LangGraph 的消息替换

### 3.6 TitleMiddleware - 标题生成

**源码位置**: `packages/harness/deerflow/agents/middlewares/title_middleware.py:22-150`

**职责**: 在首次对话后自动生成线程标题

```python
class TitleMiddleware(AgentMiddleware[TitleMiddlewareState]):
    """Automatically generate a title for the thread after the first user message."""

    def _should_generate_title(self, state: TitleMiddlewareState) -> bool:
        """Check if we should generate a title for this thread."""
        config = get_title_config()
        if not config.enabled:
            return False

        if state.get("title"):
            return False

        # 检查是否是首次完整交换 (1 用户 + 1+ AI)
        messages = state.get("messages", [])
        if len(messages) < 2:
            return False

        user_messages = [m for m in messages if m.type == "human"]
        assistant_messages = [m for m in messages if m.type == "ai"]

        return len(user_messages) == 1 and len(assistant_messages) >= 1

    def _generate_title_result(self, state: TitleMiddlewareState) -> dict | None:
        """Synchronously generate a title."""
        if not self._should_generate_title(state):
            return None

        prompt, user_msg = self._build_title_prompt(state)
        config = get_title_config()
        model = create_chat_model(name=config.model_name, thinking_enabled=False)

        try:
            response = model.invoke(prompt)
            title = self._parse_title(response.content)
            if not title:
                title = self._fallback_title(user_msg)
        except Exception:
            title = self._fallback_title(user_msg)

        return {"title": title}

    @override
    def after_model(self, state: TitleMiddlewareState, runtime: Runtime) -> dict | None:
        return self._generate_title_result(state)
```

**设计要点**:
1. **触发条件**: 首次用户消息 + 至少一次 AI 回复
2. **降级策略**: 失败时使用用户消息前 N 字符
3. **异步支持**: 同时实现同步和异步版本

## 四、中间件钩子详解

### 4.1 钩子执行时机

| 钩子 | 执行时机 | 用途 |
|------|----------|------|
| `before_agent` | Agent 执行前 | 状态初始化、路径创建 |
| `after_agent` | Agent 执行后 | 资源清理、异步任务队列 |
| `before_model` | LLM 调用前 | 上下文注入、消息预处理 |
| `after_model` | LLM 响应后 | 响应后处理、工具调用验证 |
| `wrap_tool_call` | 工具调用时 | 工具级拦截、执行流控制 |

### 4.2 同步/异步方法

```python
class MyMiddleware(AgentMiddleware[MyState]):
    @override
    def before_agent(self, state: MyState, runtime: Runtime) -> dict | None:
        # 同步版本
        return {"key": "value"}

    @override
    async def abefore_agent(self, state: MyState, runtime: Runtime) -> dict | None:
        # 异步版本
        await async_operation()
        return {"key": "value"}
```

**调用规则**:
- LangGraph 根据执行上下文选择调用同步或异步版本
- 如果只实现同步版本，异步上下文会使用同步版本

### 4.3 状态更新

```python
# 方式1: 返回字典 (推荐)
return {"messages": [new_message], "title": "new_title"}

# 方式2: 返回 None (不更新)
return None

# 方式3: 使用 Command (控制执行流)
return Command(update={"messages": [msg]}, goto=END)
```

    @override
    def after_agent(self, state: MyMiddlewareState, runtime: Runtime) -> dict | None:
        """Logic after agent execution."""

### 5.4 循环检测模板

```python
"""Loop detection template."""

import hashlib
import json
from collections import OrderedDict

class LoopDetectionMiddleware(AgentMiddleware[AgentState]):
    """Detect repetitive patterns."""

    def __init__(self, threshold: int = 3, window_size: int = 20):
        super().__init__()
        self.threshold = threshold
        self.window_size = window_size
        self._history: OrderedDict[str, list[str]] = OrderedDict()

    def _hash_pattern(self, pattern: list) -> str:
        """Create deterministic hash of a pattern."""
        blob = json.dumps(pattern, sort_keys=True, default=str)
        return hashlib.md5(blob.encode()).hexdigest()[:12]

    def _check_loop(self, thread_id: str, pattern_hash: str) -> bool:
        """Check if pattern is repeating."""
        if thread_id not in self._history:
            self._history[thread_id] = []

        history = self._history[thread_id]
        history.append(pattern_hash)
        if len(history) > self.window_size:
            history[:] = history[-self.window_size:]

        return history.count(pattern_hash) >= self.threshold
```

## 六、踩坑提醒

### 6.1 消息替换

**问题**: 直接修改消息列表可能不会触发状态更新

**解决方案**: 使用 `model_copy()` 创建新消息

```python
# 错误: 直接修改
messages[-1].content = "new content"
return {"messages": messages}

# 正确: 创建新消息
updated_msg = messages[-1].model_copy(update={"content": "new content"})
messages[-1] = updated_msg
return {"messages": messages}
```

### 6.2 异步方法实现

**问题**: 只实现同步版本，异步上下文下性能下降

**解决方案**: 同时实现同步和异步版本

```python
# 推荐: 同时实现两个版本
def before_agent(self, state, runtime):
    return self._sync_impl(state, runtime)

async def abefore_agent(self, state, runtime):
    return await self._async_impl(state, runtime)
```

### 6.3 系统消息注入

**问题**: 中间插入系统消息导致 Anthropic API 错误

**解决方案**: 使用 HumanMessage 替代 SystemMessage

```python
# 错误: 可能导致 "multiple non-consecutive system messages" 错误
return {"messages": [SystemMessage(content="warning")]}

# 正确: 使用 HumanMessage
return {"messages": [HumanMessage(content="[WARNING] ...")]}
```

### 6.4 中间件顺序依赖

**问题**: 中间件执行顺序可能影响功能

**解决方案**: 明确依赖关系并注释

```python
# 中间件注册顺序 (在 agent.py 中)
middlewares = [
    ThreadDataMiddleware(),    # 1. 首先创建目录
    UploadsMiddleware(),        # 2. 然后注入文件信息
    SandboxMiddleware(),        # 3. 然后获取沙箱
    # ... 其他中间件 ...
]
```

### 6.5 状态键冲突

**问题**: 多个中间件使用相同的状态键

**解决方案**: 使用命名空间或明确键名

```python
# 推荐: 使用前缀
return {"my_middleware_key": value}

# 或使用嵌套结构
return {"my_middleware": {"sub_key": value}}
```
