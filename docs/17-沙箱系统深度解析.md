- **所属项目**: deer-flow
- **层级位置**: `backend/packages/harness/deerflow/sandbox/`

### 图表设计解读

该链路图展示了沙箱系统的完整架构，核心设计考量如下：

1. **Provider模式**: 通过`SandboxProvider`抽象类隔离不同实现，本地和Docker可以互换，上层代码无需修改。

2. **单例复用**: `LocalSandboxProvider`使用单例模式，所有thread共享同一个本地沙箱，避免重复创建。

3. **虚拟路径系统**: AI看到的路径是虚拟的（`/mnt/user-data/*`），实际映射到主机特定目录。这提供了：
   - **一致性**: 无论主机目录结构如何，AI总是看到相同的路径
   - **隔离性**: 每个thread有独立的工作空间
   - **安全性**: AI无法访问虚拟路径之外的主机文件

4. **安全分层**:
   - **路径验证**: 拒绝`..`目录遍历
   - **安全门控**: `allow_host_bash`控制是否允许主机bash
   - **输出掩码**: 隐藏主机路径，只显示虚拟路径

5. **延迟初始化**: `ensure_sandbox_initialized()`在首次工具调用时才获取沙箱，优化性能。

### 4.1 沙箱抽象接口：`sandbox.py`

**文件路径**: `/data/deer-flow-main/backend/packages/harness/deerflow/sandbox/sandbox.py`

```python
class Sandbox(ABC):
    """Abstract base class for sandbox environments"""

    _id: str

    def __init__(self, id: str):
        self._id = id

    @property
    def id(self) -> str:
        return self._id

    @abstractmethod
    def execute_command(self, command: str) -> str:
        """Execute bash command in sandbox.

        Args:
            command: The command to execute.

        Returns:
            The standard or error output of the command.
        """
        pass

    @abstractmethod
    def read_file(self, path: str) -> str:
        """Read the content of a file.

        Args:
            path: The absolute path of the file to read.

        Returns:
            The content of the file.
        """
        pass

    @abstractmethod
    def list_dir(self, path: str, max_depth=2) -> list[str]:
        """List the contents of a directory.

        Args:
            path: The absolute path of the directory to list.
            max_depth: The maximum depth to traverse. Default is 2.

        Returns:
            The contents of the directory.
        """
        pass

    @abstractmethod
    def write_file(self, path: str, content: str, append: bool = False) -> None:
        """Write content to a file.

        Args:
            path: The absolute path of the file to write to.
            content: The text content to write to the file.
            append: Whether to append the content to the file. If False, the file will be created or overwritten.
        """
        pass

    @abstractmethod
    def update_file(self, path: str, content: bytes) -> None:
        """Update a file with binary content.

        Args:
            path: The absolute path of the file to update.
            content: The binary content to write to the file.
        """
        pass
```

#### 设计考量解读

1. **最小化接口**: 只定义5个核心方法，每个方法对应一个AI需要的能力。这简化了实现，让新的Provider可以快速开发。

2. **路径约定**: 所有路径参数都是"绝对路径"。这不是主机绝对路径，而是沙箱视角的绝对路径（如`/mnt/user-data/workspace/file.txt`）。

3. **返回值设计**:
   - `execute_command`: 返回stdout+stderr+exit code的组合字符串，简化错误处理
   - `read_file`: 返回文本内容，空文件返回空字符串
   - `list_dir`: 返回树形格式字符串，便于AI理解目录结构

4. **为什么需要`update_file`？**
   - `write_file`处理文本内容（string）
   - `update_file`处理二进制内容（bytes）
   - 分离的原因是文本文件需要编码处理（UTF-8），二进制文件直接写入

### 4.2 Provider抽象与单例：`sandbox_provider.py`

**文件路径**: `/data/deer-flow-main/backend/packages/harness/deerflow/sandbox/sandbox_provider.py`

```python
class SandboxProvider(ABC):
    """Abstract base class for sandbox providers"""

    @abstractmethod
    def acquire(self, thread_id: str | None = None) -> str:
        """Acquire a sandbox environment and return its ID.

        Returns:
            The ID of the acquired sandbox environment.
        """
        pass

    @abstractmethod
    def get(self, sandbox_id: str) -> Sandbox | None:
        """Get a sandbox environment by ID.

        Args:
            sandbox_id: The ID of the sandbox environment to retain.
        """
        pass

    @abstractmethod
    def release(self, sandbox_id: str) -> None:
        """Release a sandbox environment.

        Args:
            sandbox_id: The ID of the sandbox environment to destroy.
        """
        pass


_default_sandbox_provider: SandboxProvider | None = None


def get_sandbox_provider(**kwargs) -> SandboxProvider:
    """Get the sandbox provider singleton.

    Returns a cached singleton instance. Use `reset_sandbox_provider()` to clear
    the cache, or `shutdown_sandbox_provider()` to properly shutdown and clear.

    Returns:
        A sandbox provider instance.
    """
    global _default_sandbox_provider
    if _default_sandbox_provider is None:
        config = get_app_config()
        cls = resolve_class(config.sandbox.use, SandboxProvider)
        _default_sandbox_provider = cls(**kwargs)
    return _default_sandbox_provider
```

#### 设计考量解读

1. **生命周期方法**: Provider定义三个核心方法：
   - `acquire`: 获取/创建沙箱，返回ID
   - `get`: 根据ID获取沙箱实例
   - `release`: 释放沙箱（对于本地是无操作，对于Docker是销毁容器）

2. **thread_id参数**: `acquire`接受thread_id，允许：
   - **本地模式**: 忽略thread_id，所有thread共享沙箱
   - **Docker模式**: 每个thread有独立容器，thread_id用于标识

3. **单例模式**: `get_sandbox_provider()`使用全局变量缓存Provider实例，避免重复创建。这很重要因为：
   - Provider初始化可能很昂贵（如Docker需要拉镜像）
   - 全局单例确保所有组件使用同一个Provider

4. **动态加载**: 通过`resolve_class(config.sandbox.use)`从配置加载Provider类，支持：
   - 切换实现无需修改代码
   - 自定义Provider可以通过配置注册
   - 测试时可以注入mock Provider

5. **shutdown方法**: `shutdown_sandbox_provider()`在应用关闭时调用，确保资源正确释放（如Docker容器被销毁）。

### 4.3 本地沙箱Provider：`local_sandbox_provider.py`

**文件路径**: `/data/deer-flow-main/backend/packages/harness/deerflow/sandbox/local/local_sandbox_provider.py`

```python
_singleton: LocalSandbox | None = None


class LocalSandboxProvider(SandboxProvider):
    def __init__(self):
        """Initialize the local sandbox provider with path mappings."""
        self._path_mappings = self._setup_path_mappings()

    def _setup_path_mappings(self) -> dict[str, str]:
        """
        Setup path mappings for local sandbox.

        Maps container paths to actual local paths, including skills directory.

        Returns:
            Dictionary of path mappings
        """
        mappings = {}

        # Map skills container path to local skills directory
        try:
            from deerflow.config import get_app_config

            config = get_app_config()
            skills_path = config.skills.get_skills_path()
            container_path = config.skills.container_path

            # Only add mapping if skills directory exists
            if skills_path.exists():
                mappings[container_path] = str(skills_path)
        except Exception as e:
            # Log but don't fail if config loading fails
            logger.warning("Could not setup skills path mapping: %s", e, exc_info=True)

        return mappings

    def acquire(self, thread_id: str | None = None) -> str:
        global _singleton
        if _singleton is None:
            _singleton = LocalSandbox("local", path_mappings=self._path_mappings)
        return _singleton.id

    def get(self, sandbox_id: str) -> Sandbox | None:
        if sandbox_id == "local":
            if _singleton is None:
                self.acquire()
            return _singleton
        return None

    def release(self, sandbox_id: str) -> None:
        # LocalSandbox uses singleton pattern - no cleanup needed.
        # Note: This method is intentionally not called by SandboxMiddleware
        # to allow sandbox reuse across multiple turns in a thread.
        # For Docker-based providers (e.g., AioSandboxProvider), cleanup
        # happens at application shutdown via the shutdown() method.
        pass
```

#### 设计考量解读

1. **双层单例**:
   - **Provider单例**: `_default_sandbox_provider`确保只有一个Provider实例
   - **Sandbox单例**: `_singleton`确保只有一个LocalSandbox实例
   - **原因**: 本地沙箱不需要隔离，所有thread共享同一个文件系统访问

2. **路径映射**: `path_mappings`将容器路径（如`/mnt/skills`）映射到主机实际路径。这让AI可以使用一致的虚拟路径，而实际访问主机上的技能目录。

3. **release无操作**: 本地沙箱的`release`是空操作，因为：
   - 没有资源需要清理（不像Docker需要销毁容器）
   - 沙箱被复用，过早释放会影响性能
   - 应用关闭时自动清理

4. **配置容错**: 如果skills目录不存在或配置加载失败，记录警告但不抛异常。这提高了健壮性。

5. **为什么忽略thread_id？**
   - 本地沙箱的隔离是通过虚拟路径+thread特定目录实现的
   - 不需要为每个thread创建独立的沙箱实例
   - thread_id在路径解析时被使用，而非在acquire时

### 4.4 本地沙箱核心：`local_sandbox.py`

**文件路径**: `/data/deer-flow-main/backend/packages/harness/deerflow/sandbox/local/local_sandbox.py`

```python
class LocalSandbox(Sandbox):
    def __init__(self, id: str, path_mappings: dict[str, str] | None = None):
        """
        Initialize local sandbox with optional path mappings.

        Args:
            id: Sandbox identifier
            path_mappings: Dictionary mapping container paths to local paths
                          Example: {"/mnt/skills": "/absolute/path/to/skills"}
        """
        super().__init__(id)
        self.path_mappings = path_mappings or {}

    def _resolve_path(self, path: str) -> str:
        """
        Resolve container path to actual local path using mappings.

        Args:
            path: Path that might be a container path

        Returns:
            Resolved local path
        """
        path_str = str(path)

        # Try each mapping (longest prefix first for more specific matches)
        for container_path, local_path in sorted(self.path_mappings.items(), key=lambda x: len(x[0]), reverse=True):
            if path_str == container_path or path_str.startswith(container_path + "/"):
                # Replace the container path prefix with local path
                relative = path_str[len(container_path):].lstrip("/")
                resolved = str(Path(local_path) / relative) if relative else local_path
                return resolved

        # No mapping found, return original path
        return path_str

    def _reverse_resolve_path(self, path: str) -> str:
        """
        Reverse resolve local path back to container path using mappings.

        Args:
            path: Local path that might need to be mapped to container path

        Returns:
            Container path if mapping exists, otherwise original path
        """
        path_str = str(Path(path).resolve())

        # Try each mapping (longest local path first for more specific matches)
        for container_path, local_path in sorted(self.path_mappings.items(), key=lambda x: len(x[1]), reverse=True):
            local_path_resolved = str(Path(local_path).resolve())
            if path_str.startswith(local_path_resolved):
                # Replace the local path prefix with container path
                relative = path_str[len(local_path_resolved):].lstrip("/")
                resolved = f"{container_path}/{relative}" if relative else container_path
                return resolved

        # No mapping found, return original path
        return path_str

    def execute_command(self, command: str) -> str:
        # Resolve container paths in command before execution
        resolved_command = self._resolve_paths_in_command(command)
        shell = self._get_shell()

        if os.name == "nt":
            if self._is_powershell(shell):
                args = [shell, "-NoProfile", "-Command", resolved_command]
            elif self._is_cmd_shell(shell):
                args = [shell, "/c", resolved_command]
            else:
                args = [shell, "-c", resolved_command]

            result = subprocess.run(
                args,
                shell=False,
                capture_output=True,
                text=True,
                timeout=600,
            )
        else:
            result = subprocess.run(
                resolved_command,
                executable=shell,
                shell=True,
                capture_output=True,
                text=True,
                timeout=600,
            )
        output = result.stdout
        if result.stderr:
            output += f"\nStd Error:\n{result.stderr}" if output else result.stderr
        if result.returncode != 0:
            output += f"\nExit Code: {result.returncode}"

        final_output = output if output else "(no output)"
        # Reverse resolve local paths back to container paths in output
        return self._reverse_resolve_paths_in_output(final_output)
```

#### 设计考量解读

1. **双向路径解析**:
   - **_resolve_path**: 容器路径→主机路径（执行前转换）
   - **_reverse_resolve_path**: 主机路径→容器路径（输出时转换）
   - **原因**: AI应该看到虚拟路径，不应该知道主机目录结构

2. **最长前缀匹配**: 按路径长度降序排序，确保更具体的映射优先匹配。这避免了短路径误匹配长路径前缀的问题。

3. **跨平台Shell检测**: `_get_shell()`自动检测可用shell：
   - **Linux/Mac**: zsh → bash → sh
   - **Windows**: PowerShell → cmd.exe
   - **Fallback**: 抛出明确错误，指导用户安装

4. **命令路径转换**: `_resolve_paths_in_command()`使用正则表达式匹配并替换命令中的所有容器路径。这支持`cat /mnt/skills/README.md`这样的命令。

5. **输出掩码**: `_reverse_resolve_paths_in_output()`将输出中的主机路径替换回虚拟路径，避免泄露主机目录结构。

6. **超时保护**: subprocess设置600秒超时，防止命令无限期运行。

### 4.5 虚拟路径系统：`tools.py`

**文件路径**: `/data/deer-flow-main/backend/packages/harness/deerflow/sandbox/tools.py`

```python
VIRTUAL_PATH_PREFIX = "/mnt/user-data"

def replace_virtual_path(path: str, thread_data: ThreadDataState | None) -> str:
    """Replace virtual /mnt/user-data paths with actual thread data paths.

    Mapping:
        /mnt/user-data/workspace/* -> thread_data['workspace_path']/*
        /mnt/user-data/uploads/* -> thread_data['uploads_path']/*
        /mnt/user-data/outputs/* -> thread_data['outputs_path']/*

    Args:
        path: The path that may contain virtual path prefix.
        thread_data: The thread data containing actual paths.

    Returns:
        The path with virtual prefix replaced by actual path.
    """
    if thread_data is None:
        return path

    mappings = _thread_virtual_to_actual_mappings(thread_data)
    if not mappings:
        return path

    # Longest-prefix-first replacement with segment-boundary checks.
    for virtual_base, actual_base in sorted(mappings.items(), key=lambda item: len(item[0]), reverse=True):
        if path == virtual_base:
            return actual_base
        if path.startswith(f"{virtual_base}/"):
            rest = path[len(virtual_base):].lstrip("/")
            return _join_path_preserving_style(actual_base, rest)

    return path


def _resolve_and_validate_user_data_path(path: str, thread_data: ThreadDataState) -> str:
    """Resolve a /mnt/user-data virtual path and validate it stays in bounds.

    Returns the resolved host path string.
    """
    resolved_str = replace_virtual_path(path, thread_data)
    resolved = Path(resolved_str).resolve()
    _validate_resolved_user_data_path(resolved, thread_data)
    return str(resolved)


def _validate_resolved_user_data_path(resolved: Path, thread_data: ThreadDataState) -> None:
    """Verify that a resolved host path stays inside allowed per-thread roots.

    Raises PermissionError if the path escapes workspace/uploads/outputs.
    """
    allowed_roots = [
        Path(p).resolve()
        for p in (
            thread_data.get("workspace_path"),
            thread_data.get("uploads_path"),
            thread_data.get("outputs_path"),
        )
        if p is not None
    ]

    if not allowed_roots:
        raise SandboxRuntimeError("No allowed local sandbox directories configured")

    for root in allowed_roots:
        try:
            resolved.relative_to(root)
            return
        except ValueError:
            continue

    raise PermissionError("Access denied: path traversal detected")
```

#### 设计考量解读

1. **三层虚拟路径**:
   - **/mnt/user-data/workspace**: 临时工作目录
   - **/mnt/user-data/uploads**: 用户上传文件
   - **/mnt/user-data/outputs**: 最终交付文件
   - **原因**: 分离不同用途的文件，便于管理和权限控制

2. **ThreadData传递**: thread_data包含thread_id对应的实际路径，由ThreadDataMiddleware注入到state中。这确保每个thread有独立的文件空间。

3. **双重验证**:
   - **_reject_path_traversal**: 拒绝`..`段
   - **_validate_resolved_user_data_path**: 使用`relative_to`确保解析后的路径在允许根目录内
   - **原因**: 防止路径遍历攻击（如`../../../etc/passwd`）

4. **最长前缀优先**: 按虚拟路径长度降序处理，确保`/mnt/user-data/workspace`在`/mnt/user-data`之前匹配，避免错误替换。

5. **路径风格保留**: `_join_path_preserving_style`保留原始路径的分隔符风格（/或\），确保Windows兼容性。

```python
def uses_local_sandbox_provider(config=None) -> bool:
    """Return True when the active sandbox provider is the host-local provider."""
    if config is None:
        config = get_app_config()

    sandbox_cfg = getattr(config, "sandbox", None)
    sandbox_use = getattr(sandbox_cfg, "use", "")
    if sandbox_use in _LOCAL_SANDBOX_PROVIDER_MARKERS:
        return True
    return sandbox_use.endswith(":LocalSandboxProvider") and "deerflow.sandbox.local" in sandbox_use


def is_host_bash_allowed(config=None) -> bool:
    """Return whether host bash execution is explicitly allowed."""
    if config is None:
        config = get_app_config()

    sandbox_cfg = getattr(config, "sandbox", None)
    if sandbox_cfg is None:
        return True
    if not uses_local_sandbox_provider(config):
        return True
    return bool(getattr(sandbox_cfg, "allow_host_bash", False))
```

#### 设计考量解读

1. **默认安全**: `allow_host_bash`默认为False，本地沙箱的bash执行被禁用。这是因为：
   - 本地沙箱不是真正的隔离边界
   - bash命令可以访问整个主机文件系统
   - 恶意AI可能执行危险命令

2. **明确的错误消息**: 当bash被禁用时，返回详细的错误消息：
   - 说明为什么禁用
   - 提供解决方案（切换到AioSandboxProvider）
   - 警告风险（仅在完全可信的环境启用）

3. **Provider检测**: `uses_local_sandbox_provider()`通过检查配置中的类路径来判断是否使用本地Provider。这支持：
   - 完整类路径：`deerflow.sandbox.local:LocalSandboxProvider`
   - 短类名：`:LocalSandboxProvider`（当模块名已知时）

4. **分层门控**:
   - **工具层**: `bash_tool`检查`is_host_bash_allowed()`
   - **子代理层**: bash子代理检查`LOCAL_BASH_SUBAGENT_DISABLED_MESSAGE`
   - **原因**: 多层防御确保安全策略一致

```python
@tool("bash", parse_docstring=True)
def bash_tool(runtime: ToolRuntime[ContextT, ThreadState], description: str, command: str) -> str:
    """Execute a bash command in a Linux environment.


    - Use `python` to run Python code.
    - Prefer a thread-local virtual environment in `/mnt/user-data/workspace/.venv`.
    - Use `python -m pip` (inside the virtual environment) to install Python packages.

    Args:
        description: Explain why you are running this command in short words. ALWAYS PROVIDE THIS PARAMETER FIRST.
        command: The bash command to execute. Always use absolute paths for files and directories.
    """
    try:
        sandbox = ensure_sandbox_initialized(runtime)
        if is_local_sandbox(runtime):
            if not is_host_bash_allowed():
                return f"Error: {LOCAL_HOST_BASH_DISABLED_MESSAGE}"
            ensure_thread_directories_exist(runtime)
            thread_data = get_thread_data(runtime)
            validate_local_bash_command_paths(command, thread_data)
            command = replace_virtual_paths_in_command(command, thread_data)
            command = _apply_cwd_prefix(command, thread_data)
            output = sandbox.execute_command(command)
            return mask_local_paths_in_output(output, thread_data)
        ensure_thread_directories_exist(runtime)
        return sandbox.execute_command(command)
    except SandboxError as e:
        return f"Error: {e}"
    except PermissionError as e:
        return f"Error: {e}"
    except Exception as e:
        return f"Error: Unexpected error executing command: {_sanitize_error(e, runtime)}"
```

#### 设计考量解读

1. **description参数**: 要求AI解释为什么执行命令。这：
   - 强制AI思考命令目的
   - 提供更好的可观测性
   - 帮助用户理解AI行为

2. **本地沙箱特殊处理**: 当检测到本地沙箱时，执行额外步骤：

   - **路径验证**: `validate_local_bash_command_paths()`检查命令中的路径

   - **工作目录前缀**: `_apply_cwd_prefix()`添加`cd`命令
   - **输出掩码**: `mask_local_paths_in_output()`隐藏主机路径

3. **目录创建**: `ensure_thread_directories_exist()`在首次使用时创建目录，延迟创建提升性能。

4. **错误分类**: 捕获不同类型的异常并返回用户友好的错误消息：
   - `SandboxError`: 沙箱特定错误

### 4.8 沙箱中间件：`middleware.py`

**文件路径**: `/data/deer-flow-main/backend/packages/harness/deerflow/sandbox/middleware.py`

```python
class SandboxMiddleware(AgentMiddleware[SandboxMiddlewareState]):
    """Create a sandbox environment and assign it to an agent.

    Lifecycle Management:
    - With lazy_init=True (default): Sandbox is acquired on first tool call
    - With lazy_init=False: Sandbox is acquired on first agent invocation (before_agent)
    - Sandbox is reused across multiple turns within the same thread
    - Sandbox is NOT released after each agent call to avoid wasteful recreation
    - Cleanup happens at application shutdown via SandboxProvider.shutdown()
    """

    state_schema = SandboxMiddlewareState

    def __init__(self, lazy_init: bool = True):
        """Initialize sandbox middleware.

        Args:
            lazy_init: If True, defer sandbox acquisition until first tool call.
                      If False, acquire sandbox eagerly in before_agent().
                      Default is True for optimal performance.
        """
        super().__init__()
        self._lazy_init = lazy_init

    def _acquire_sandbox(self, thread_id: str) -> str:
        provider = get_sandbox_provider()
        sandbox_id = provider.acquire(thread_id)
        logger.info(f"Acquiring sandbox {sandbox_id}")
        return sandbox_id

    @override
    def before_agent(self, state: SandboxMiddlewareState, runtime: Runtime) -> dict | None:
        # Skip acquisition if lazy_init is enabled
        if self._lazy_init:
            return super().before_agent(state, runtime)

        # Eager initialization (original behavior)
        if "sandbox" not in state or state["sandbox"] is None:
            thread_id = (runtime.context or {}).get("thread_id")
            if thread_id is None:
                return super().before_agent(state, runtime)
            sandbox_id = self._acquire_sandbox(thread_id)
            logger.info(f"Assigned sandbox {sandbox_id} to thread {thread_id}")
            return {"sandbox": {"sandbox_id": sandbox_id}}
        return super().before_agent(state, runtime)

    @override
    def after_agent(self, state: SandboxMiddlewareState, runtime: Runtime) -> dict | None:
        sandbox = state.get("sandbox")
        if sandbox is not None:
            sandbox_id = sandbox["sandbox_id"]
            logger.info(f"Releasing sandbox {sandbox_id}")
            get_sandbox_provider().release(sandbox_id)
            return None

        if (runtime.context or {}).get("sandbox_id") is not None:
            sandbox_id = runtime.context.get("sandbox_id")
            logger.info(f"Releasing sandbox {sandbox_id} from context")
            get_sandbox_provider().release(sandbox_id)
            return None

        # No sandbox to release
        return super().after_agent(state, runtime)
```

#### 设计考量解读

1. **延迟初始化（lazy_init=True）**:
   - **默认行为**: 沙箱在首次工具调用时获取
   - **优势**: 避免不必要的资源分配（如Docker容器创建）
   - **适用**: 大多数对话不需要工具调用

2. **急切初始化（lazy_init=False）**:
   - **行为**: 在`before_agent`中立即获取沙箱
   - **适用**: 需要沙箱就绪的场景（如预加载依赖）

3. **沙箱复用**: 中间件不在`after_agent`中释放沙箱，让沙箱在多个turn中复用。这显著提升性能，因为：
   - 本地沙箱：避免重复初始化
   - Docker沙箱：避免重复创建容器

4. **状态存储**: 沙箱ID存储在`state["sandbox"]["sandbox_id"]`中，这确保：
   - 后续中间件可以看到沙箱ID
   - 工具可以通过`ensure_sandbox_initialized()`获取沙箱

5. **为什么`after_agent`还保留release逻辑？**
   - 虽然默认不释放，但某些场景可能需要急切释放
   - 保持接口完整性，支持不同使用模式

## 5. 底层设计思想

### 5.1 整体设计理念：Provider模式 + 虚拟路径系统

**为什么采用Provider模式？**

1. **实现无关性**: 上层代码（工具、中间件）只依赖`Sandbox`接口，不关心具体实现。

2. **可互换性**: 本地沙箱和Docker沙箱可以无缝切换，只需修改配置。

3. **测试友好**: 可以注入Mock Provider进行单元测试，无需真实沙箱。

**为什么需要虚拟路径系统？**

1. **一致性**: AI总是看到相同的路径结构，无论主机目录如何组织。

2. **隔离性**: 每个thread有独立的`/mnt/user-data/`，互不干扰。

3. **安全性**: AI只能访问虚拟路径下的文件，无法访问主机其他部分。

4. **可移植性**: 配置和代码可以在不同机器上运行，无需修改路径。

### 5.2 核心痛点解决

**痛点1: AI执行命令可能破坏主机**

- **问题**: 直接执行bash命令可以删除任意文件
- **解决**:
  - 默认禁用本地沙箱的bash
  - 路径验证拒绝`..`
  - 输出掩码隐藏主机路径

**痛点2: 不同thread的文件互相干扰**

- **问题**: 多个用户同时使用，文件可能冲突
- **解决**: 每个thread有独立的workspace/uploads/outputs目录

**痛点3: AI无法访问技能目录**

- **问题**: 技能文件位于主机特定目录，AI不知道路径
- **解决**: 通过`path_mappings`将`/mnt/skills`映射到实际技能目录

**痛点4: 主机路径泄露暴露系统结构**

- **问题**: 错误消息和输出包含主机绝对路径
- **解决**: 输出掩码将主机路径替换回虚拟路径

**痛点5: 沙箱创建开销大**

- **问题**: 每次对话创建Docker容器很慢
- **解决**: 延迟初始化+沙箱复用，只在需要时创建

### 5.3 行业对比优势

| 特性 | DeerFlow | OpenAI Code Interpreter | E2B |
|------|----------|----------------------|-----|
| 本地执行 | 支持（可选） | 不支持 | 支持 |
| 虚拟路径 | 统一/mnt/user-data | 无 | 无 |
| 路径验证 | 双重验证 | 容器隔离 | 容器隔离 |
| 输出掩码 | 自动掩码 | N/A | N/A |
| 技能目录集成 | 无缝集成 | 不支持 | 不支持 |
| 延迟初始化 | 支持 | 不支持 | 不支持 |

### 5.4 扩展性设计

1. **自定义Provider**: 继承`SandboxProvider`，实现`acquire/get/release`方法。可以集成：
   - Kubernetes Pod
   - AWS Lambda
   - 远程服务器

2. **自定义路径映射**: 在Provider初始化时添加`path_mappings`，支持任意目录映射。

3. **自定义安全策略**: 通过`allow_host_bash`等配置控制安全级别。

4. **自定义工具**: 基于`ensure_sandbox_initialized()`模式，可以创建新的沙箱工具。

### 5.5 设计取舍

1. **本地 vs Docker**:
   - **选择**: 两者都支持，默认本地
   - **权衡**: 本地方便但不安全，Docker安全但复杂
   - **原因**: 开发阶段用本地，生产环境用Docker

2. **延迟 vs 急切初始化**:
   - **选择**: 默认延迟（lazy_init=True）
   - **权衡**: 延迟提升性能，急切简化逻辑
   - **原因**: 大多数对话不需要工具调用

3. **路径掩码 vs 原始输出**:
   - **选择**: 自动掩码
   - **权衡**: 掩码隐藏细节，原始输出便于调试
   - **原因**: 安全性优先，调试可以通过日志

4. **单例 vs 多实例**:
   - **选择**: 本地沙箱单例，Docker沙箱多实例
   - **权衡**: 单例简单，多实例隔离更好
   - **原因**: 本地不需要隔离，Docker需要

```python
import re

def mask_paths(output: str, mapping: dict[str, str]) -> str:
    """掩码输出中的路径。"""
    result = output
    for actual, virtual in sorted(mapping.items(), key=lambda x: len(x[0]), reverse=True):
        escaped = re.escape(actual)
        pattern = re.compile(escaped + r"(?:/[^\s\"';&|<>()]*)?")
        result = pattern.sub(lambda m: virtual + m.group(0)[len(actual):], result)
```

**关键要点**:

```python
from pathlib import Path

class PathValidator:
    """路径验证器，防止目录遍历。"""

    def __init__(self, allowed_roots: list[str]):
        self.allowed_roots = [Path(r).resolve() for r in allowed_roots]

    def validate(self, path: str) -> None:
        """验证路径在允许根目录内。"""
        # 拒绝显式遍历
        if ".." in path.replace("\\", "/").split("/"):
            raise PermissionError("Path traversal detected")

        # 验证解析后的路径
        resolved = Path(path).resolve()
        for root in self.allowed_roots:
            try:
                resolved.relative_to(root)
                return  # 在允许根目录内
            except ValueError:
                continue

        raise PermissionError("Access denied")
```

### 7.4 输出掩码工具

```python
import re
from pathlib import Path

class OutputMasker:
    """输出掩码器，隐藏主机路径。"""

    def __init__(self, mappings: dict[str, str]):
        """
        Args:
            mappings: {主机路径: 虚拟路径} 的映射
        """
        self.mappings = mappings

    def mask(self, output: str) -> str:
        """掩码输出中的主机路径。"""
        result = output
        # 按主机路径长度降序处理（长路径优先）
        for host_path, virtual_path in sorted(self.mappings.items(),
                                             key=lambda x: len(x[0]),
                                             reverse=True):
            host_resolved = str(Path(host_path).resolve())
            escaped = re.escape(host_resolved).replace(r"\\", r"[/\\]")
            pattern = re.compile(escaped + r"(?:[/\\][^\s\"';&|<>()]*)?")

            def replace(match):
                matched = match.group(0)
                relative = matched[len(host_resolved):].lstrip("/\\")
                return f"{virtual_path}/{relative}" if relative else virtual_path

            result = pattern.sub(replace, result)
        return result
```
