威胁模型与防护

威胁风险防护层机制
未授权 API 访问 🔴 高 L2 认证层 JWT Token 签发/验证，Token 过期自动失效
跨站请求伪造 🔴 高 L2 认证层 CSRF Token 双重提交校验
恶意工具调用 🔴 高 L3 策略层 Guardrail Allow/Deny 规则，工具白名单过滤
路径遍历攻击 🔴 高 L4 执行层 虚拟文件系统 /mnt/ 前缀，路径映射 + 输出遮蔽
危险 Bash 命令 🟡 中 L4 执行层 SandboxAuditMiddleware 命令审计和过滤
SKILL.md 注入攻击 🟡 中 L5 扫描层 LLM 模型扫描 Prompt 注入、角色覆盖、权限提升
Agent 无限循环 🟡 中 L5 扫描层 LoopDetection 连续重复调用检测，达到 hard_limit 强制终止
Token 消耗攻击 🟢 低 运行时 TokenUsageMiddleware 实时统计，Summarization 自动压缩
LLM 服务雪崩 🟢 低 运行时 Circuit Breaker 熔断器，连续失败自动熔断 + 恢复

L2 认证层 — JWT + CSRF

sequenceDiagram
    participant C as Client
    participant CORS as CORS
    participant CSRF as CSRF
    participant JWT as JWT Auth
    participant API as API Handler

    C->>CORS: 1. Origin 检查
    CORS-->>CSRF: 通过
    CSRF->>CSRF: 2. CSRF Token 校验
    CSRF-->>JWT: 通过
    JWT->>JWT: 3. Token 解签/验证
    JWT-->>API: 认证通过 + user_id
    API->>API: 4. 权限校验
🔑

JWT Token

签发带 user_id 的 JWT Token。每次请求验证签名和过期时间。支持多用户会话管理，Token 过期自动失效需重新登录。

🛡️

CSRF 保护

双重提交 CSRF Token 机制。前端从 Cookie 提取 Token，在请求 Header 中携带。服务端比对 Cookie 和 Header 中的 Token 一致性。

🌐

CORS 策略

配置允许的 Origin 白名单。生产环境仅开放前端域名，开发环境支持 localhost。Nginx 层 + FastAPI 层双重 CORS 检查。

🛡️

🚫

GuardrailMiddleware

全局工具策略中间件。通过 config.yaml 配置 AllowlistProvider 或自定义 Provider，定义 allowed/denied 工具列表。每个工具调用前检查。

# config.yaml
guardrails:
  enabled: true
  provider:
    use: ...AllowlistProvider
    config:
      denied_tools: ["bash"]
📋

DeferredToolFilter

SKILL.md 中的 allowed-tools 字段创建工具白名单。一旦任何技能声明了限制，则进入白名单模式，只能使用列出的工具。不可通过嵌套技能降级权限。

🔒

🏠

Local Sandbox 路径映射

Agent 看到的路径以 /mnt/ 开头，底层映射到宿主机真实目录。文件操作被限制在映射范围内，防止路径遍历到 /etc/passwd 等敏感文件。

🐳

Docker Sandbox 容器隔离

每次执行在独立 Docker 容器中进行。容器有独立文件系统和网络命名空间。/mnt/ 挂载点映射到工作目录。容器生命周期自动管理。

🔍

SandboxAuditMiddleware

Bash 命令审计中间件。在执行前检查命令内容，过滤危险操作（如 rm -rf /）。记录命令执行日志用于安全审计和事后追溯。

🎭

输出遮蔽

工具返回结果中的敏感路径（如宿主机真实路径）自动替换为沙箱虚拟路径。防止 Agent 通过返回值推断宿主机目录结构。

🔎

SKILL.md 安全扫描

使用可配置的 LLM 模型对 SKILL.md 内容进行安全扫描。检测四大类威胁：Prompt 注入、系统角色覆盖尝试、权限提升指令、数据外泄通道。

决策模式：allow (通过) / warn (警告但加载) / block (阻止加载)。扫描失败时采用保守策略直接阻止。

🔁

LoopDetection 循环防护

检测 Agent 是否陷入无限循环（连续重复相同的工具调用）。达到 warn_threshold 时发出警告，达到 hard_limit 时强制终止执行，防止 Token 消耗耗尽。

# config.yaml
loop_detection:
  enabled: true
  warn_threshold: 3  # 连续 3 次相同调用 → 警告
  hard_limit: 5      # 连续 5 次 → 强制终止
✅

安全最佳实践

🔑

密钥管理

永远不要将 API Key 硬编码在 config.yaml 中。使用 $ENV_VAR 引用环境变量。生产环境使用密钥管理服务（如 Vault / AWS Secrets Manager）。

🐳

生产用 Docker 沙箱

生产环境务必使用 Docker Sandbox Provider。Local Sandbox 仅适合开发调试。配置 allow_host_bash: false 禁止直接宿主机执行。

🛡️

启用 Guardrail

对涉及敏感操作的 Agent，启用 Guardrail 并配置 denied_tools 列表。对技能使用 allowed-tools 白名单限制可用工具范围。

📊

监控与审计

启用 LangSmith/Langfuse 追踪，监控异常 Token 消耗模式。定期审查 SandboxAudit 日志。设置 Token 预算上限。
