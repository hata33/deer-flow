Step 1: 请求接入

用户通过 Web UI 或 IM 平台发送消息。前端通过 POST /api/threads/{id}/runs 发起请求，经 Nginx 反向代理到达 FastAPI 网关

Step 2: 认证校验

请求依次通过 CORS → CSRF → JWT 认证中间件链。验证 Token 有效性、用户权限，确保请求合法

Step 3: 运行创建

RunManager 创建运行实例，初始化 RunJournal (回调日志)、EventStore (事件存储)、StreamBridge (流桥接)

Step 4: Agent 执行

LeadAgent 执行中间件链 → LLM 流式调用 → 工具执行循环。每个 Token 通过 StreamBridge 实时推送到前端 SSE

Step 5: 完成清理

Agent 执行完毕，持久化运行记录到数据库，关闭 SSE 流，发送 [DONE] 信号，前端渲染最终响应

🌉

StreamBridge 流桥接

后端核心组件。使用有界缓冲区(bounded buffer)将 LangChain 回调事件桥接到 SSE 协议格式，防止内存溢出

🔀

三源合并

将 LLM Token 流、工具执行事件、Agent 状态变更三个事件源归一化处理，统一编码为 SSE 格式推送

🎨

前端实时渲染

前端通过 EventSource API 接收流，实时更新：状态指示器、对话消息、文件/Artifact 变更

📋

优先级装配

工具按优先级依次装配：配置文件工具 → MCP 外部工具 → 内置工具 → 子代理工具。同名工具高优先级覆盖低优先级

🛡️

安全防线

每次工具调用经过 Guardrail 安全策略（allow/deny 规则）和 Sandbox 审计（危险命令过滤）双重检查

🔒

沙箱隔离

文件操作和命令执行在沙箱中进行。本地沙箱通过路径映射和输出遮蔽实现安全，Docker 沙箱提供完整容器隔离

📥

L1 队列层 - 收集与去重

所有消息来源（用户、Agent、工具）进入队列层。通过 Hash 去重防止重复记忆，优先级排序确保重要信息优先处理

💾

L2 存储层 - 持久化与索引

记忆以 JSON 文件形式存储，利用文件修改时间(mtime)实现高效缓存。构建倒排索引用于快速检索

🎯

L3 注入层 - 选择与格式化

相关性选择器根据当前上下文筛选记忆，窗口控制确保不超过 Token 预算，格式化为 LLM 可消费的结构化上下文

💎

L4 使用层 - 提取与注入

从记忆中提取关键事实，结合 BM25 和语义排序计算相关性，应用时间衰减函数，最终注入到 System Prompt 中
