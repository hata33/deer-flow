快速启动

Step 1 — 克隆仓库

git clone https://github.com/your-org/deer-flow.git
cd deer-flow

Step 2 — 复制配置文件

# 主配置
cp config.example.yaml config.yaml

# MCP 扩展配置
cp extensions_config.example.json extensions_config.json

Step 3 — 编辑 config.yaml

至少配置一个模型（填写 API Key）：

models:
  - name: gpt-4
    use: langchain_openai:ChatOpenAI
    model: gpt-4
    api_key: $OPENAI_API_KEY  # 设置环境变量

Step 4 — 设置环境变量

export OPENAI_API_KEY="sk-your-key"
export JWT_SECRET="your-random-secret"

Step 5 — 启动服务

# 开发模式 (一键启动)
make dev

# 或手动启动后端
cd backend && uv run uvicorn app.main:app --port 8001 --reload

# 手动启动前端
cd frontend && pnpm dev

Nginx 反向代理

核心 Nginx 配置

# nginx.conf 关键配置
server {
    listen 2026;
    server_name your-domain.com;

    # SSL (生产推荐)
    # listen 443 ssl;
    # ssl_certificate /path/to/cert.pem;
    # ssl_certificate_key /path/to/key.pem;

    # 前端静态资源 / SSR
    location / {
        proxy_pass http://nextjs:3000;
        proxy_set_header Host $host;
    }

    # API 网关
    location /api/ {
        proxy_pass http://fastapi:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # SSE 流式推送 (关键配置)
    location /api/threads/ {
        proxy_pass http://fastapi:8001;
        proxy_buffering off;          # 禁用缓冲
        proxy_cache off;              # 禁用缓存
        proxy_read_timeout 600s;      # 长连接超时
        proxy_set_header Connection '';
        proxy_http_version 1.1;
        chunked_transfer_encoding off;
    }
}

数据库选型

📁

SQLite (开发)

零配置，文件存储在 .deer-flow/data/。适合单机开发和测试。无需额外容器。

database:
  backend: sqlite
  sqlite_dir: .deer-flow/data
🐘

PostgreSQL (生产)

推荐生产环境使用。支持并发连接、事务、JSON 查询。通过 Docker Compose 或外部 RDS 部署。

database:
  backend: postgresql
  url: $DATABASE_URL
# postgresql://user:pass@host:5432/deerflow
📊

可观测性 — 追踪与监控

🔬

LangSmith

LangChain 生态原生追踪平台。可视化 Agent 执行链路、Token 消耗、工具调用详情。支持 Trace 对比和版本追踪。

🔭

Langfuse

开源 LLM 可观测性平台。自部署或云服务。支持 Prompt 版本管理、A/B 测试、成本分析、用户行为追踪。

📈

内置指标

TokenUsageMiddleware 实时统计每次运行的 Token 消耗。RunJournal 记录完整事件日志。EventStore 存储运行元数据供后续分析。

⚡

熔断器

Circuit Breaker 监控 LLM 调用失败率。连续失败达阈值自动熔断，保护后端不被持续重试拖垮。恢复超时后自动半开探测。

🔧

常见故障排查

❌ Agent 响应为空或报 LLM 调用失败

检查 config.yaml 中模型 api_key 是否正确设置（环境变量是否生效）。确认模型 use 路径正确。查看 log_level: debug 获取详细错误日志。检查网络是否能访问 LLM API 端点。

❌ SSE 流式推送中断 / 前端收不到实时消息

检查 Nginx 是否配置了 proxy_buffering off 和 proxy_cache off。确认 proxy_read_timeout 足够长（建议 600s+）。检查 proxy_http_version 1.1 配置。

❌ Docker Sandbox 无法启动 / 权限错误

确认 Docker daemon 正在运行：docker ps。检查容器挂载目录权限。Docker-in-Docker 场景需挂载 /var/run/docker.sock。生产环境确认 sandbox.use 指向 DockerSandboxProvider。

❌ 技能加载失败 / SKILL.md 解析报错

检查 SKILL.md frontmatter 格式（name 和 description 为必填字段）。YAML 语法错误会导致解析失败。安全扫描可能阻止了加载，检查 log_level: debug 中的扫描结果。

❌ Agent 陷入循环 / Token 消耗异常

检查 loop_detection 是否启用。调整 hard_limit（默认 5）到更低值。检查 Prompt 是否给了 Agent 明确的停止条件。启用 Langfuse 追踪分析循环原因。

❌ 数据库连接失败

SQLite: 确认 sqlite_dir 路径可写。PostgreSQL: 确认 $DATABASE_URL 正确，数据库已创建，用户有读写权限。Docker Compose 场景确认数据库容器健康：docker-compose ps。

❌ MCP Server 连接失败

检查 extensions_config.json 中 enabled 是否为 true。stdio 类型确认命令路径正确。sse/http 类型确认 URL 可达。检查 env 中的环境变量是否设置。

❌ JWT 认证失败 / 401 错误

确认 $JWT_SECRET 在前后端一致。检查 Token 是否过期。前端确认 Token 存储和发送逻辑正确。CORS 配置确认允许前端域名。
