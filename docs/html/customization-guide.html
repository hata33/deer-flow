Extension Points Detail

通过系统 Prompt 模板和SOUL.md 人格文件定制 Agent 的行为方式和沟通风格。

系统 Prompt 模板占位符

模板位于 agents/lead_agent/prompt.py，以下占位符会被动态替换：

# 系统提示词模板中的可注入占位符
{agent_name}       # → Agent 名称 (默认 "DeerFlow 2.0")
{soul}           # → SOUL.md 人格描述内容
{skills_section}  # → 已加载技能列表
{subagent_section}# → 子代理系统指令
{self_update_section}# → 自更新指令 (自定义 Agent 专属)

SOUL.md 人格文件

放在 Agent 配置目录下，定义 Agent 的个性、沟通风格和行为偏好：

SOUL.md
# 我的月度简报助手

## 性格
你是一位专业的数据分析顾问，善于从海量数据中提炼关键洞察。

## 沟通风格
- 使用结构化的列表和表格呈现信息
- 数据引用始终标注来源和时间范围
- 结论先行，支撑数据紧随其后
- 适当使用 emoji 标记关键指标趋势 (📈 📉 ➡️)

## 行为偏好
- 每次分析前先明确数据时间范围
- 自动对比上月数据并计算环比变化
- 识别异常波动时主动标注并给出可能原因

框架内置 19 个中间件，用户可继承 AgentMiddleware 添加自定义处理逻辑。

Middleware 基类 — 4 个可实现的钩子

Python
from deerflow.agents.middleware import AgentMiddleware
from deerflow.models import ThreadState

class MyMiddleware(AgentMiddleware[ThreadState]):
    state_schema = ThreadState

    def before_agent(self, state, config):
        """Agent 执行前：上下文注入"""
        return state

    def wrap_model_call(self, state, request, config, *, call_next):
        """LLM 调用包装：错误处理/重试"""
        return call_next(state, request, config)

    def wrap_tool_call(self, state, request, config, *, call_next):
        """工具调用包装：安全审计"""
        return call_next(state, request, config)

    def after_agent(self, state, config):
        """Agent 执行后：后处理"""
        return state

注册位置

使用装饰器精确控制中间件在链中的位置：

Python
@Next(SandboxMiddleware)
class MyMiddleware(...):
    # 插入到 SandboxMiddleware 之后
    ...

@Prev(LoopDetectionMiddleware)
class MyMiddleware(...):
    # 插入到 LoopDetection 之前
    ...

内置中间件一览 (19个)

按执行顺序：ThreadData → Uploads → Sandbox → DanglingToolCall → Guardrail → ToolErrorHandling → LLMErrorHandling → SandboxAudit → Summarization → DynamicContext → Todo → TokenUsage → Title → Memory → ViewImage → DeferredToolFilter → SubagentLimit → LoopDetection → Clarification

在 skills/custom/ 下创建目录，编写 SKILL.md 即可注册新技能。

SKILL.md 格式

SKILL.md
---
name: monthly-report
description: "月度数据分析与简报生成技能"
allowed-tools:
  - web_search
  - read_file
  - bash
---

# 月度简报生成技能

## 工作流程
1. 从用户指定的数据源采集当月数据
2. 对比上月数据，计算环比变化
3. 识别关键趋势和异常指标
4. 生成结构化简报 (摘要 + 指标表 + 趋势分析 + 建议)

## 输出格式
- 每个指标标注 📈 上升 / 📉 下降 / ➡️ 持平
- 异常数据用 ⚠️ 标记
- 最终以 Markdown 表格形式输出

Python

@tool("fetch_monthly_data")

    month: str

    return query_data(source, month)

通过 extensions_config.json 配置外部 MCP Server，扩展工具能力。

MCP Server 配置格式

传输类型

stdio — 本地子进程通信
sse — Server-Sent Events 远程
http — HTTP 请求远程

Step 3 — 自定义 Tools

用 Python @tool 定义数据采集和导出工具，注册到 config.yaml。

Python
@tool("fetch_monthly_data")
def fetch_monthly_data(metric: str, month: str) -> str:
    """获取指定指标的月度数据"""
    return query_database(metric, month)

@tool("export_report")
def export_report(content: str, filename: str) -> str:
    """导出简报为 Markdown 文件"""
    return write_file(filename, content)

Step 4 — 编写 SKILL.md

创建 skills/custom/monthly-report/SKILL.md，定义技能行为和工具限制。

SKILL.md
---
name: monthly-report
description: "月度数据分析与简报生成"
allowed-tools:
  - fetch_monthly_data
  - export_report
  - read_file
  - bash
---

# 月度简报生成技能

## 工作流程
1. 确认用户需要的指标和时间范围
2. 使用 fetch_monthly_data 采集每个指标
3. 对比上月数据，计算环比
4. 识别 Top 3 增长和 Top 3 下降指标
5. 生成结构化简报：摘要 → 指标表 → 趋势分析 → 行动建议
6. 使用 export_report 导出文件

## 输出规范
- 📈 上升 >5%  · 📉 下降 >5%  · ➡️ 波动 <5%
- ⚠️ 标记异常值
- 表格使用 Markdown 格式

Step 5 — SOUL.md 人格定义

创建 .agent/SOUL.md，定义数据分析师角色。

SOUL.md
# 月度简报数据分析师

## 角色
你是公司月度业务数据分析专家，擅长从多维数据中提炼洞察。

## 风格
- 结论先行，数据支撑紧跟其后
- 使用表格呈现指标对比
- 自动计算环比变化
- 关键发现用加粗标注

## 约束
- 所有数据引用必须标注来源
- 缺失数据时主动提示而非编造
- 日期格式统一使用 YYYY-MM-DD

Step 6 — 配置子代理

定义一个 data-collector 子代理专门负责数据采集，与主 Agent 分工协作。

config.yaml
subagents:
  custom_agents:
    data-collector:
      description: "数据采集专员：从各数据源采集月度指标"
      system_prompt: |
        你是数据采集专员。你的任务是：
        1. 从数据库和文件中采集指定月度的业务指标
        2. 清洗数据，处理缺失值
        3. 返回结构化 JSON 格式的数据
        不要分析数据，只负责采集和清洗。
      tools:
        - bash
        - read_file
        - fetch_monthly_data
      skills:
        - monthly-report
      max_turns: 40
      timeout_seconds: 300

Step 7 — 可选中间件增强

  enabled: true
