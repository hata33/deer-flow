# 【31】示例工作流与技能使用指南

## 1. 模块全局定位

- **所属项目**：deer-flow
- **层级位置**：`skills/public/`、`skills/custom/`、`skills/public/*/SKILL.md`
- **核心作用**：提供开箱即用的AI工作流模板，涵盖数据分析、图像生成、PPT制作、深度研究等常见场景
- **业务价值**：作为"最佳实践知识库"，将复杂任务的标准流程封装为可复用技能，显著提升AI代理执行质量与一致性
- **设计初衷**：设计用于解决"AI代理输出质量不稳定"问题——通过技能模板指导代理遵循专家级工作流程，确保结果可预测、高质量

## 3. 核心技能深度解析

### 3.1 深度研究（deep-research）

**文件路径**：`/data/deer-flow-main/skills/public/deep-research/SKILL.md`

**核心方法论**：

```
Phase 1: 广泛探索
├── 初始调查：主话题概览
├── 识别维度：关键子主题、角度、方面
└── 映射领域：不同视角、利益相关者

Phase 2: 深入研究
├── 精确查询：针对性关键词搜索
├── 多种表达：尝试不同关键词组合
├── 获取完整内容：使用web_fetch读取全文
└── 追踪引用：搜索提到的其他资源

Phase 3: 多样化验证
├── 事实与数据
├── 案例与实例
├── 专家观点
├── 趋势与预测
├── 对比分析
└── 挑战与批评

Phase 4: 综合检查
└── 质量清单验证
```

**使用示例**：

```
用户：帮我研究一下2024年AI在医疗领域的应用

代理工作流：
1. 加载deep-research技能
2. Phase 1搜索：
   - "AI healthcare applications 2024"
   - "artificial intelligence medical diagnosis"
   - "healthcare AI market trends"
3. Phase 2深入研究：
   - "AI radiology FDA approved systems"
   - "chest X-ray AI detection accuracy"
4. Phase 3多样化：
   - 搜索统计数据、案例研究、专家观点
5. 输出：综合性研究报告
```

**设计亮点**：
- **时间感知**：使用`<current_date>`获取当前日期，确保搜索时效性
- **质量门槛**：定义6个质量检查点，确保研究深度
- **迭代优化**：研究是迭代的，初步结果后识别差距继续深入

### 3.2 图像生成（image-generation）

**文件路径**：`/data/deer-flow-main/skills/public/image-generation/SKILL.md`

**核心流程**：

```
1. 理解需求
   ├── 主题/内容：图像中应包含什么
   ├── 风格偏好：艺术风格、情绪、色调
   ├── 技术规格：宽高比、构图、光照
   └── 参考图像：风格/构图指导图像

2. 创建结构化提示词
   └── JSON格式：characters、style、composition、lighting、color_palette

3. 执行生成
   └── python /mnt/skills/public/image-generation/scripts/generate.py
```

**JSON提示词模板**：

```json
{
  "characters": [{
    "gender": "female",
    "age": "mid-20s",
    "ethnicity": "Japanese",
    "clothing": "stylish trench coat, designer handbag"
  }],
  "style": "Leica M11 street photography, film-like rendering",
  "composition": "medium shot, rule of thirds, shallow depth of field",
  "lighting": "neon lights from signs, wet pavement reflections"
}
```

**设计亮点**：
- **结构化提示**：JSON格式确保提示词可解析、可复用
- **参考图像支持**：通过`--reference-images`参数提供视觉指导
- **宽高比控制**：支持16:9、4:3、1:1等多种比例
- **负向提示词**：明确排除不希望出现的元素

**结合image_search工具**：

```
工作流：
1. 先使用image_search搜索参考图
   image_search(query="Japanese woman street photography 1990s")
2. 下载参考图到本地
3. 使用参考图生成：
   python generate.py --reference-images /mnt/user-data/uploads/ref1.jpg ...
```

### 3.3 数据分析（data-analysis）

**文件路径**：`/data/deer-flow-main/skills/public/data-analysis/SKILL.md`

**核心能力**：

- **文件结构检查**：schema、columns、types、row counts
- **SQL查询执行**：支持任意SQL查询
- **统计摘要**：mean、median、stddev、percentiles
- **多表支持**：多文件/多sheet联合查询
- **结果导出**：CSV、JSON、Markdown格式

**使用示例**：

```
用户上传sales_2024.xlsx并要求分析

代理工作流：
1. 检查文件结构
   python analyze.py --files /mnt/user-data/uploads/sales_2024.xlsx --action inspect

2. 统计摘要
   python analyze.py --files ... --action summary --table Sheet1

3. SQL查询
   python analyze.py --files ... --action query --sql "SELECT category, SUM(amount) as total FROM Sheet1 GROUP BY category"

4. 导出结果
   python analyze.py --files ... --action query --sql "..." --output-file /mnt/user-data/outputs/report.csv
```

**设计亮点**：
- **DuckDB引擎**：列式分析数据库，支持复杂SQL与窗口函数
- **缓存机制**：SHA256哈希缓存文件解析结果，重复查询接近即时
- **表命名规则**：Excel sheet名称直接作为表名，CSV文件名（无扩展名）作为表名
- **特殊字符处理**：空格自动替换为下划线，数字开头表名用双引号

## 4. 技能使用模式

### 4.1 触发条件

代理通过`<skill_system>`中的技能描述自动判断何时加载技能：

```
<skill>
  <name>deep-research</name>
  <description>Use this skill instead of WebSearch for ANY question requiring web research...</description>
  <location>/mnt/skills/public/deep-research/SKILL.md</location>
</skill>
```

**触发时机**：
- 用户请求匹配技能描述中的关键词
- 技能明确说明适用场景（如"when user asks X"）
- 技能声明优于其他工具（如"instead of WebSearch"）

### 4.2 渐进式加载

**三步流程**：

1. **识别需求**：代理分析用户请求，匹配相关技能
2. **读取技能**：`read_file`加载SKILL.md内容
3. **遵循指令**：按技能定义的工作流执行

**示例**：

```
用户："帮我做个市场分析报告"

代理思考过程：
1. 匹配到consulting-analysis技能
2. read_file("/mnt/skills/public/consulting-analysis/SKILL.md")
3. 按技能中的框架步骤执行：
   - Step 1: Clarify Objectives
   - Step 2: Gather Context
   - Step 3: Apply Framework
   - Step 4: Generate Deliverables
```

### 4.3 多技能协作

**场景组合**：

```
用户："创建一个关于AI趋势的PPT，需要最新数据"

代理技能组合：
1. deep-research → 收集2024年AI趋势数据
2. chart-visualization → 制作趋势图表
3. ppt-generation → 生成最终PPT
```

**协作模式**：
- **顺序执行**：一个技能的输出作为下一个的输入
- **并行执行**：通过subagent并发调用多个技能
- **结果合成**：主代理汇总各技能输出

## 5. 技能开发指南

### 5.1 创建新技能

**步骤1：初始化结构**

```bash
cd skills/public
python /mnt/skills/public/skill-creator/scripts/init_skill.py --name my-skill --category public
```

**步骤2：编写SKILL.md**

```markdown
---
name: my-skill
description: Brief description of when to use this skill
---

# My Skill Title

## Overview
Explain what this skill does...

## When to Use
- Scenario 1
- Scenario 2

## Workflow
### Step 1: ...
### Step 2: ...

## Common Mistakes to Avoid
- ❌ Mistake 1
- ✅ Correct approach
```

**步骤3：添加脚本**

```bash
mkdir -p scripts
# 添加Python脚本，确保可执行
chmod +x scripts/script.py
```

**步骤4：本地测试**

```bash
python /mnt/skills/public/skill-creator/scripts/run_eval.py --skill-name my-skill
```

**步骤5：打包分发**

```bash
python /mnt/skills/public/skill-creator/scripts/package_skill.py --skill-name my-skill
# 生成my-skill.skill文件
```

### 5.2 技能验证规则

**Frontmatter必需字段**：
- `name`：技能名称（hyphen-case，小写字母+连字符+数字）
- `description`：技能描述（1-1024字符，不含`<`或`>`）

**可选字段**：
- `license`：许可证信息
- `allowed-tools`：依赖的工具列表
- `metadata`：扩展元数据（version、author等）

**验证命令**：

```python
from deerflow.skills.validation import _validate_skill_frontmatter
from pathlib import Path

is_valid, message, name = _validate_skill_frontmatter(Path("skills/public/my-skill"))
```

## 6. 最佳实践

### 6.1 技能设计原则

1. **单一职责**：每个技能专注一个明确场景
2. **清晰触发条件**：description中明确说明适用场景
3. **可操作步骤**：工作流步骤具体、可执行
4. **错误处理**：常见错误与解决方案
5. **示例驱动**：提供完整示例展示用法

### 6.2 提示词编写技巧

1. **明确指令**：使用祈使句，直接告诉代理做什么
2. **分步说明**：复杂任务分解为可执行步骤
3. **代码示例**：提供可拷贝的命令示例
4. **注意事项**：使用`[!NOTE]`强调重要细节
5. **模板引用**：复杂结构使用模板文件

### 6.3 脚本开发规范

1. **参数验证**：检查必需参数，提供友好错误信息
2. **路径处理**：使用绝对路径，支持虚拟路径映射
3. **输出管理**：输出到`/mnt/user-data/outputs/`便于展示
4. **错误处理**：捕获异常，记录详细日志

## 8. 技能生态

### 8.1 公共技能（skills/public/）

**特点**：
- 版本控制，随项目分发
- 官方向导，开箱即用
- 覆盖常见场景（数据、图像、PPT等）

### 8.2 自定义技能（skills/custom/）

**特点**：
- 用户私有，不提交到仓库
- 支持通过Gateway API或.skill归档安装
- 可覆盖公共技能或添加新能力

### 8.3 技能分发

**安装方式**：

1. **手动安装**：将技能目录复制到`skills/custom/`
2. **归档安装**：通过`POST /api/skills/install`上传.skill文件
3. **Git克隆**：克隆包含技能的仓库到custom目录
