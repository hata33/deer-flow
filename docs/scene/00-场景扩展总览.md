# 场景扩展总览

DeerFlow 支持通过工具+技能+中间件的模式扩展到不同业务场景。以下是已规划的扩展方案。

---

## 已规划场景

| 场景 | 目录 | 状态 | 核心改动 |
|------|------|------|---------|
| NL2SQL | [NL2SQL/](NL2SQL/) | 方案已完成 | 6 个 SQL 工具 + Schema 中间件 + 安全校验 |
| Agentic RAG | [Agentic-RAG/](Agentic-RAG/) | 方案已完成 | 6 个检索工具 + 向量存储 + 混合检索 |

---

## 扩展模式

所有场景遵循统一的扩展模式：

```
场景 = 工具集 + 技能定义 + 中间件 + 配置
```

```
community/{场景}/tools.py        → 工具实现
skills/custom/{场景}/SKILL.md    → 技能定义（Prompt + 工具白名单）
agents/middlewares/{场景}_*.py   → 场景专用中间件
config/{场景}_config.py          → 配置模型
```

### 添加新场景的步骤

1. **在 `community/` 下创建工具模块**
2. **在 `skills/custom/` 下创建技能目录和 SKILL.md**
3. **按需创建场景专用中间件**
4. **在 `config.yaml` 中添加配置**
5. **编写测试和文档**

## 扩展点详解

### 工具扩展

```python
# community/{场景}/tools.py
from langchain_core.tools import tool

@tool(parse_docstring=True)
def my_tool(param: str) -> str:
    """工具描述（Agent 靠这个描述决定何时使用）"""
    # 实现
```

注册方式：
```yaml
# config.yaml
tools:
  - name: my_tool
    use: deerflow.community.{场景}.tools:my_tool
```

### 技能扩展

```markdown
<!-- skills/custom/{场景}/SKILL.md -->
---
name: {场景名}
description: "场景描述"
allowed-tools:
  - tool_a
  - tool_b
---

## 技能指令
...
```

### 中间件扩展

```python
# agents/middlewares/{场景}_middleware.py
from deerflow.agents.middlewares.base import AgentMiddleware

class MyMiddleware(AgentMiddleware[ThreadState]):
    def before_model(self, state, config):
        # 在 LLM 调用前注入上下文
        return state
```
