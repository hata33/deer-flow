# 通道集成

**问题**: Agent 不能只活在 Web 界面里——用户可能在飞书、Slack、Discord 等平台上工作，需要把 Agent 能力带到用户所在的地方。

---

## 问题 1：什么是通道（Channel）？

通道是 Agent 与外部通信平台的桥接器：

```
飞书用户 → 飞书 Channel Adapter → ChannelManager → Agent → 回复到飞书
Slack 用户 → Slack Channel Adapter → ChannelManager → Agent → 回复到 Slack
Web 用户  → HTTP SSE (Gateway)    → RunManager    → Agent → SSE 推送
```

每个平台一个 Adapter，统一接入 ChannelManager。

---

## 问题 2：支持的通道类型？

| 通道 | 通信方式 | 特点 |
|------|---------|------|
| Web (Gateway) | HTTP + SSE | 内置，默认通道 |
| 飞书 | WebSocket / Webhook | 企业办公场景 |
| Slack | Socket Mode | 海外团队 |
| 自定义 | 实现适配器接口 | 任意平台 |

```yaml
channels:
  feishu:
    enabled: true
    app_id: "${FEISHU_APP_ID}"
    app_secret: "${FEISHU_APP_SECRET}"
  slack:
    enabled: false
    bot_token: "${SLACK_BOT_TOKEN}"
    app_token: "${SLACK_APP_TOKEN}"
```

## 问题 3：ChannelManager 怎么路由消息？

消息总线（MessageBus）模式：

```
飞书消息 → MessageBus.put(topic, message)
                │
                ▼ MessageBus 路由
            根据 topic 分发
                │
                ▼ 对应的处理器
            Agent 处理 → 结果返回对应通道
```

Topic 路由规则支持群聊场景：
- 群聊中只有 @机器人 的消息才触发 Agent
- 私聊中所有消息都触发
- 不同群可以绑定不同的 Agent 配置

---

## 问题 4：通道消息怎么转换？

每个 Adapter 负责双向转换：

```
外部消息 → 统一内部格式
    │
    ▼ 内部格式
{
    "user_id": "external_user_123",
    "channel": "feishu",
    "thread_id": "auto_generated",
    "content": "帮我分析这个文件",
    "attachments": [{"name": "report.pdf", "url": "..."}]
}
    │
    ▼ Agent 处理
    │
    ▼ Agent 回复
"分析结果如下：..."
    │
    ▼ 转换为飞书消息格式
{
    "msg_type": "text",
    "content": {"text": "分析结果如下：..."}
}
```

## 问题 5：流式回复怎么在通道中展示？

Web 用 SSE 逐字渲染，但飞书/Slack 不支持：

| 通道 | 流式策略 |
|------|---------|
| Web (SSE) | 逐 token 推送，前端实时渲染 |
| 飞书 | 等 Agent 完成后一次性发送（或分块更新） |
| Slack | 同上 |

```python
# 非流式通道的适配
if channel.supports_streaming:
    await stream_to_channel(agent_output)
else:
    full_response = await collect_full_response(agent_output)
    await channel.send_message(full_response)
```

## 问题 6：ChannelService 的生命周期？

```
启动阶段:
    │
    ▼ 读取配置，确定启用的通道
    │
    ▼ 创建 ChannelManager
    │
    ▼ 初始化各 Adapter
    ├── 飞书 Adapter → WebSocket 连接
    ├── Slack Adapter → Socket Mode 连接
    └── ...
    │
    ▼ 注册消息处理器
    │
    ▼ 开始监听消息

运行阶段:
    接收消息 → 路由 → Agent 处理 → 回复

关闭阶段:
    通知所有 Adapter 断开连接
    等待进行中的请求完成
    清理资源
```

## 问题 7：URL 怎么解析？

Agent 需要知道自己的 URL（用于回调、OAuth 回调等）：

```
URL 解析优先级:
1. config.yaml 中显式配置
2. 环境变量
3. 默认值（localhost）
```

```yaml
channels:
  base_url: "https://agent.example.com"  # 最高优先级
```

---

## 问题 8：群聊场景怎么处理？

群聊比私聊复杂——不是所有消息都应该触发 Agent：

```python
# 路由规则
routing_rules:
  - type: "mention"       # 被 @时响应
    trigger: "@deerbot"
  - type: "keyword"       # 关键词触发
    trigger: "/agent"
  - type: "reply"         # 回复 Bot 消息时
    trigger: "reply_to_bot"
```

Agent 的回复也只在群中可见（不是广播给所有人）。

---

## 数据流概览

```
飞书用户发送 @deerbot 帮我分析代码
    │
    ▼ 飞书 Adapter 接收
    │
    ▼ 转换为内部消息格式
    │
    ▼ MessageBus 路由
    │
    ▼ ChannelManager 调度到 Agent
    │
    ▼ Agent 执行（与其他通道共享同一套中间件）
    │
    ▼ Agent 回复
    │
    ▼ 转换为飞书消息格式
    │
    ▼ 飞书 Adapter 发送
```
