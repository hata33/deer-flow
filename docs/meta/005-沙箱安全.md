# 沙箱安全

**问题**: Agent 能执行 bash 命令和读写文件，如果没有安全隔离，可能导致系统文件被篡改、敏感数据泄露或路径穿越攻击。

---

## 问题 1：沙箱隔离了什么？

Agent 看到的是虚拟文件系统，实际操作被映射到安全的物理路径：

```
Agent 视角                    物理路径
/mnt/user-data/workspace  →  .deer-flow/users/{user_id}/threads/{thread_id}/user-data/workspace
/mnt/skills               →  skills/  (只读)
/mnt/shared               →  .deer-flow/shared/  (只读)
```

Agent 无法感知物理路径，所有操作都在映射后的安全区域内。

---

## 问题 2：路径穿越攻击怎么防？

`validate_local_tool_path` 函数做三层检查：

```python
def validate_local_tool_path(path, thread_data, *, read_only=False):
    # 1. 路径穿越检查
    _reject_path_traversal(path)  # 拒绝 ..、//、符号链接逃逸

    # 2. 路径族匹配
    if path.startswith("/mnt/user-data/"):
        return  # 用户数据区，允许
    elif path.startswith("/mnt/skills/"):
        if read_only:
            return  # 只读访问技能区
        raise PermissionError("Skills directory is read-only")

    # 3. 默认拒绝
    raise PermissionError(f"Path not in allowed families: {path}")
```

## 问题 3：bash 命令能逃出沙箱吗？

`local_sandbox` 对 bash 执行做了限制：

- **工作目录锁定**：命令在映射后的工作区内执行
- **环境变量隔离**：只注入必要的虚拟路径变量
- **禁止危险命令**：`rm -rf /`、`dd`、`mkfs` 等在 guardrails 层被拦截

```yaml
sandbox:
  use: "local"
  allow_host_bash: false  # 禁止直接在宿主机执行
```

---

## 问题 4：只读挂载怎么实现的？

每个挂载点有 `read_only` 标志：

```python
@dataclass(frozen=True)
class PathMapping:
    container_path: str    # 虚拟路径
    local_path: str        # 物理路径
    read_only: bool        # 是否只读
```

写入操作（`write_file`、`str_replace`）在执行前检查目标路径的挂载点是否为只读，是则拒绝。

---

## 问题 5：不同用户的沙箱互相隔离吗？

是的。每个线程有独立的沙箱实例，路径映射包含 `user_id` 和 `thread_id`：

```
用户 A / 线程 1:  .deer-flow/users/a/threads/1/user-data/workspace
用户 B / 线程 2:  .deer-flow/users/b/threads/2/user-data/workspace
```

用户 A 的 Agent 无法访问用户 B 的任何文件。

---

## 问题 6：还有更强的隔离方案吗？

有。除了默认的 `local` 沙箱，还支持 `aio` 沙箱（基于 Docker 容器）：

```yaml
sandbox:
  use: "aio"  # Docker 容器隔离
```

| 维度 | local | aio |
|------|-------|-----|
| 隔离级别 | 文件系统路径映射 | 完整容器隔离 |
| 性能 | 快（无容器开销） | 较慢（需启动容器） |
| 安全性 | 依赖路径校验 | 内核级隔离 |
| 适用场景 | 开发/测试 | 生产/多租户 |

---

## 问题 7：沙箱操作有审计日志吗？

有。`sandbox_audit_middleware` 记录所有文件操作：

```
[audit] user=a thread=1 tool=write_file path=/mnt/user-data/workspace/main.py
[audit] user=a thread=1 tool=bash command="python main.py"
```

日志包含用户、线程、工具、路径/命令等关键信息，用于安全回溯。

---

## 问题 8：如何配置挂载点？

```yaml
sandbox:
  mounts:
    - path: /mnt/user-data/workspace
      local_path: .deer-flow/users/{user_id}/threads/{thread_id}/user-data/workspace
      read_only: false
    - path: /mnt/skills
      local_path: skills
      read_only: true
    - path: /mnt/shared
      local_path: .deer-flow/shared
      read_only: true
```

`{user_id}` 和 `{thread_id}` 是运行时替换的模板变量。

## 数据流概览

```
Agent 调用 write_file("/mnt/user-data/workspace/app.py", content)
    │
    ▼ sandbox_middleware 拦截
路径校验: validate_local_tool_path
    ├── 检查路径穿越 → 通过
    ├── 检查路径族 → /mnt/user-data/ 允许写入
    └── 检查只读标志 → 非只读，允许
    │
    ▼ 路径映射
/mnt/user-data/workspace/app.py → .deer-flow/users/a/threads/1/user-data/workspace/app.py
    │
    ▼ sandbox_audit_middleware 记录
    │
    ▼ 写入物理文件
```
