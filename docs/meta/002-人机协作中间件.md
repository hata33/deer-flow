# 人机协作中间件

**问题**: Agent 执行过程中需要向用户请求澄清、确认高风险操作，否则可能因信息不足或操作失误导致严重后果。

---

## 问题 1：什么时候 Agent 需要找用户？

五种场景触发澄清：

| 类型 | 含义 | 示例 |
|------|------|------|
| `missing_info` | 缺少关键信息 | "你想部署到哪个环境？" |
| `ambiguous_requirement` | 需求模糊 | "你说的'优化'是指速度还是内存？" |
| `approach_choice` | 多种方案需选择 | "用 Redis 还是用内存缓存？" |
| `risk_confirmation` | 高风险操作确认 | "即将删除生产数据库，确认？" |
| `suggestion` | Agent 有建议 | "建议用 TypeScript 重写，你觉得？" |

---

## 问题 2：澄清机制怎么实现的？

Agent 调用内置工具 `ask_clarification`，该工具通过 LangGraph 的 `Command(interrupt=...)` 中断执行流，将问题推送给前端。用户回答后，回答作为新消息注入对话，Agent 继续执行。

```
Agent 调用 ask_clarification("请选择方案")
    │
    ▼ Command(interrupt=value)
执行暂停，前端收到中断事件
    │
    ▼ 用户回答
回答注入对话，Agent 恢复执行
```

关键：这不是简单的 `input()`，而是利用 LangGraph 的检查点机制——中断时状态完整保存，恢复时无缝续接。

---

## 问题 3：如何防止 Agent 滥用澄清？

`clarification_middleware` 在 `after_tool` 钩子中拦截 `ask_clarification` 的调用，做两层校验：

1. **频率限制**：同一轮对话中调用次数有上限
2. **上下文校验**：检查是否有实际工具结果可供参考，避免无意义提问

---

## 问题 4：澄清和护栏（Guardrails）有什么区别？

两个独立的机制，解决不同问题：

| 维度 | 澄清（Clarification） | 护栏（Guardrails） |
|------|----------------------|-------------------|
| 触发方 | Agent 主动 | 系统策略强制 |
| 目的 | 获取信息或确认 | 控制工具访问权限 |
| 用户交互 | 必须等用户回答 | 用户可能无感知 |
| 粒度 | 对话级 | 工具调用级 |

---

## 问题 5：前端怎么展示澄清消息？

前端通过消息类型区分渲染方式：

- `clarification` 类型消息 → 渲染为卡片，展示问题 + 回答区域
- 回答后 → 卡片折叠，显示 "已回答"

`frontend/src/core/messages/utils.ts` 中根据消息的 `name` 字段和 `additional_kwargs` 识别澄清类型，选择对应的 UI 组件。

---

## 问题 6：中断期间 Agent 的状态怎么保存？

LangGraph 的检查点机制自动保存。中断时当前完整的图状态（包括已执行步骤、待执行节点、消息历史）写入检查点。用户回答后，从检查点恢复，Agent 从中断点继续。

```
中断前: [节点A完成] → [节点B执行中] → 中断
                                      ↓
                              检查点保存完整状态
                                      ↓
用户回答:                        从检查点恢复
                                      ↓
继续:    [节点B完成] → [节点C] → ...
```

---

## 问题 7：多轮澄清怎么处理？

每次澄清都是独立的中断-恢复循环。Agent 可以连续调用多次 `ask_clarification`，每次都走完整的暂停→保存→恢复流程。但 `clarification_middleware` 会限制单轮中的总次数，防止无限循环。

---

## 数据流概览

```
Agent 执行中遇到不确定点
    │
    ▼ 调用 ask_clarification
clarification_middleware 校验
    │
    ▼ 校验通过
Command(interrupt=clarification_value)
    │
    ▼ LangGraph 检查点保存
前端收到中断事件，渲染澄清卡片
    │
    ▼ 用户回答
回答注入对话，从检查点恢复
    │
    ▼ Agent 继续执行
```
