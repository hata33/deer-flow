# 线程目录与隔离

**问题**: 多用户多线程并发使用时，每个线程的工作空间（文件、上传、沙箱）必须完全隔离，否则用户 A 的 Agent 能看到用户 B 的文件。

---

## 问题 1：每个线程的目录结构是什么？

```
.deer-flow/
    └── users/
        └── {user_id}/
            ├── memory.json              # 用户级记忆
            ├── agents/
            │   └── {agent_name}/
            │       └── memory.json      # Agent 级记忆
            └── threads/
                └── {thread_id}/
                    ├── metadata.json    # 线程元数据
                    └── user-data/
                        ├── workspace/   # 工作区（可读写）
                        └── uploads/     # 上传文件（可读写）
```

每个层级有明确的职责和隔离边界。

---

## 问题 2：目录什么时候创建？

`thread_data_middleware` 在首次 Run 时自动创建：

```
线程首次使用
    │
    ▼ thread_data_middleware 检查
    │
    ▼ 目录不存在 → 创建完整目录树
    │
    ▼ 注入 thread_data 到上下文
```

延迟创建——不预先创建所有线程的目录，而是按需创建。

---

## 问题 3：用户隔离怎么实现？

路径中包含 `user_id`：

```
用户 A 的文件: .deer-flow/users/user_a/threads/thread_1/user-data/
用户 B 的文件: .deer-flow/users/user_b/threads/thread_2/user-data/
```

沙箱的路径映射确保 Agent 只能看到自己的虚拟路径：

```
Agent 视角: /mnt/user-data/workspace
实际路径: .deer-flow/users/user_a/threads/thread_1/user-data/workspace
```

Agent 无法构造路径访问其他用户的数据——路径映射在中间件层完成。

---

## 问题 4：ThreadMetaStore 是什么？

线程元数据的存储接口：

```python
class ThreadMetaStore(Protocol):
    async def get(self, thread_id: str) -> ThreadMeta | None
    async def put(self, thread_id: str, meta: ThreadMeta) -> None
    async def delete(self, thread_id: str) -> None
    async def list_threads(self, user_id: str) -> list[ThreadMeta]
```

两种实现：
- **SQLite/PostgreSQL**：生产使用，支持持久化和查询
- **内存**：测试使用，速度快但不持久

## 问题 5：线程 ID 有什么限制？

```python
# 模式验证
pattern = r"^[A-Za-z0-9_\-]+$"
```

只允许字母、数字、下划线和连字符。这防止了路径注入攻击——不能用 `../` 等特殊字符。

---

## 问题 6：线程目录和沙箱的关系？

线程目录是沙箱路径映射的**源数据**：

```
线程目录:
  .deer-flow/users/a/threads/1/user-data/workspace
      │
      ▼ 沙箱映射配置
mounts:
  - path: /mnt/user-data/workspace
    local_path: .deer-flow/users/a/threads/1/user-data/workspace
    read_only: false
      │
      ▼ Agent 视角
Agent 访问 /mnt/user-data/workspace → 物理路径映射
```

没有线程目录，沙箱就不知道把虚拟路径映射到哪里。

---

## 问题 7：线程删除时目录会清理吗？

线程删除时，对应的文件目录会被清理。但 `ThreadMeta` 中的元数据保留一段时间（可配置），用于审计和恢复。

---

## 问题 8：Agent 级别的记忆怎么存储？

每个 Agent 可以有独立的记忆文件：

```
users/{user_id}/memory.json              # 全局记忆
users/{user_id}/agents/research/memory.json  # 研究 Agent 专属记忆
users/{user_id}/agents/bash/memory.json      # Bash Agent 专属记忆
```

主 Agent 使用全局记忆，子 Agent 使用专属记忆。这防止不同 Agent 的记忆互相污染。

---

## 数据流概览

```
Run 请求到达（user_id=a, thread_id=1）
    │
    ▼ thread_data_middleware
    │
    ▼ 检查目录是否存在
    ├── 不存在 → 创建目录树
    └── 存在 → 读取 thread_data
    │
    ▼ 注入 thread_data 到上下文
    │
    ▼ 沙箱使用 thread_data 配置路径映射
    │
    ▼ Agent 在隔离环境中执行
```
