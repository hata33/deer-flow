# 文件上传流程

**问题**: 用户上传文档（PDF、Word、PPT、Excel）让 Agent 分析，需要安全接收、格式转换、注入 Agent 上下文，且上传完成后不污染长期记忆。

---

## 问题 1：支持哪些文件格式？

| 格式 | 转换方式 | 输出 |
|------|---------|------|
| PDF | pymupdf4llm 或 markitdown | Markdown |
| Word (.docx) | python-docx → 文本 | Markdown |
| PPT (.pptx) | python-pptx → 文本 | Markdown |
| Excel (.xlsx) | openpyxl → 文本 | Markdown |
| 图片 | 视觉模型描述 | 文本描述 |
| 纯文本 | 直接读取 | 原文 |

转换目标统一为 Markdown——LLM 理解 Markdown 效率最高。

---

## 问题 2：上传 API 怎么调用？

```
POST /api/threads/{thread_id}/uploads
Content-Type: multipart/form-data

文件字段: files (支持多文件)
```

响应返回文件元数据列表，包含虚拟路径和文件摘要。

---

## 问题 3：文件存储在哪里？

上传的文件存储在线程专属目录：

```
物理路径: .deer-flow/users/{user_id}/threads/{thread_id}/user-data/uploads/
虚拟路径: /mnt/user-data/uploads/   （Agent 视角）
```

每个 Thread 的上传文件互相隔离。Agent 通过虚拟路径引用文件，物理路径不可见。

---

## 问题 4：文件内容怎么注入 Agent 上下文？

`uploads_middleware` 在 `before_model` 钩子中将文件内容注入：

```xml
<uploaded_files>
  <file name="report.pdf" path="/mnt/user-data/uploads/report.pdf">
    <outline>
      第 1 章: 概述
      第 2 章: 数据分析
      ...
    </outline>
    <content>
      [Markdown 格式的文档内容]
    </content>
  </file>
</uploaded_files>
```

不是把整个文件塞进去——而是先提供大纲（outline），再提供内容（content）。大文件可能只注入前 N 页的预览。

---

## 问题 5：大文件怎么处理？

| 策略 | 触发条件 | 效果 |
|------|---------|------|
| 大纲 + 预览 | 文件超过 token 预算 | Agent 先看大纲，按需读取详细内容 |
| 分页注入 | PDF 超过页数限制 | 只注入前 N 页，后续通过工具按页读取 |
| 摘要注入 | 极大文件 | LLM 先生成文件摘要，注入摘要而非原文 |

```yaml
uploads:
  max_file_size: 50MB
  max_preview_pages: 10
  max_injection_tokens: 4000
```

---

## 问题 6：上传文件会进入长期记忆吗？

**不会**。`memory_updater` 在提取记忆时会主动清理上传相关的内容：

```python
# 记忆提取时剥离上传文件引用
cleaned_summary = strip_upload_references(memory_summary)
```

为什么：上传文件是临时的上下文信息，不应成为长期记忆的一部分。否则记忆会被文件名和文档细节污染。

---

## 问题 7：文件安全怎么保证？

三道防线：

1. **沙箱隔离**：文件只能通过虚拟路径访问，不能逃出线程目录
2. **格式校验**：上传时检查文件扩展名和 MIME 类型
3. **大小限制**：单文件最大 50MB（可配置）

```yaml
uploads:
  allowed_extensions: [".pdf", ".docx", ".pptx", ".xlsx", ".txt", ".md", ".png", ".jpg"]
  max_file_size: 50MB
```

---

## 问题 8：上传过程中 Agent 能继续工作吗？

能。上传是独立的 HTTP 请求，不阻塞 Agent 执行。上传完成后，下次 LLM 调用时 `uploads_middleware` 自动将新文件注入上下文。

时序：
```
1. 用户上传文件 → POST /uploads → 返回成功
2. Agent 继续当前任务
3. 下次 LLM 调用 → uploads_middleware 检测到新文件 → 注入上下文
4. Agent 看到文件内容，开始分析
```

---

## 数据流概览

```
用户上传 report.pdf
    │
    ▼ HTTP POST /api/threads/{id}/uploads
    │
    ▼ 格式校验 + 大小检查
    │
    ▼ PDF → Markdown 转换
    │
    ▼ 存储到线程 uploads 目录
    │
    ▼ 下次 LLM 调用前
uploads_middleware 注入
    ├── 文件大纲
    └── 文件预览（或全文）
    │
    ▼ Agent 分析文件内容
    │
    ▼ 对话结束
记忆更新时剥离上传引用
```
