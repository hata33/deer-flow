# 循环检测机制

**问题**: LLM 有时会陷入重复调用同一工具、反复执行相同命令的死循环，白白消耗 token 和时间，永远无法完成任务。

---

## 问题 1：Agent 死循环长什么样？

典型模式：

```
第 5 轮: bash("npm test")  → 失败
第 6 轮: bash("npm test")  → 失败（完全相同的命令）
第 7 轮: bash("npm test")  → 失败
第 8 轮: bash("npm test")  → 失败
...
```

LLM 不是在"思考"——它在机械重复。不干预的话会耗尽 `recursion_limit`（通常 25-50 轮）。

---

## 问题 2：为什么 LLM 会循环？

几个常见原因：

| 原因 | 示例 |
|------|------|
| 错误信息误导 | LLM 读不懂报错，以为再试一次就行 |
| 参数没变化 | 忘记修改工具参数，重复相同输入 |
| 上下文过长 | 记不住已经试过了什么 |
| 模型能力不足 | 小模型更容易陷入循环 |

---

## 问题 3：怎么检测循环？

双重检测机制，覆盖不同模式：

**1. 哈希检测（精确重复）**

对每轮的 `(tool_name, tool_input)` 计算哈希，在滑动窗口内统计重复次数：

```
窗口大小 = 20 轮
重复 3 次 → 警告（追加提示到 AIMessage）
重复 5 次 → 强制停止（移除 tool_calls）
```

检测的是精确重复——工具名和参数完全一致。

**2. 频率检测（同一工具过度使用）**

统计同一工具在窗口内的调用频率：

```
bash 调用 30 次 → 警告
bash 调用 50 次 → 强制停止
其他工具 30 次 → 警告
其他工具 50 次 → 强制停止
```

不要求参数相同——只要 bash 被调了太多次就算。

---

## 问题 4：警告和强停具体做什么？

**警告（Warning）**：

```python
# 在 AIMessage 末尾追加提示
ai_message.content += "\n\n[系统警告] 检测到你可能陷入循环，" \
    "请尝试不同的方法。"
```

LLM 看到警告后通常会调整策略。这是"柔性干预"。

**强停（Hard Stop）**：

```python
# 移除所有 tool_calls，设置结束标记
ai_message.tool_calls = []
ai_message.additional_kwargs["finish_reason"] = "stop"
```

Agent 被迫停止工具调用，以文本形式返回当前进展。这是"硬干预"。

---

## 问题 5：线程级别的跟踪怎么实现？

循环检测是线程级别的——每个 Thread 独立统计：

```python
# LRU 缓存，最多跟踪 100 个 Thread
_thread_history = LRUCache(maxsize=100)
```

不同 Thread 的调用历史互不干扰。LRU 保证长时间不活跃的 Thread 自动清理。

---

## 问题 6：和 recursion_limit 有什么区别？

| 维度 | 循环检测 | recursion_limit |
|------|---------|----------------|
| 检测能力 | 识别重复模式 | 只计总数 |
| 干预方式 | 警告→强停，有过渡 | 直接截断 |
| 上下文感知 | 是（分析调用模式） | 否 |
| 触发时机 | 更早（3-5 次重复） | 更晚（25-50 轮） |

`recursion_limit` 是 LangGraph 的兜底——如果循环检测没拦住，最终 recursion_limit 会强制停止。

---

## 问题 7：子 Agent 的循环怎么控制？

子 Agent 有独立的循环检测，但阈值更严格：

- 子 Agent 的 `recursion_limit` 更低（按配置）
- `subagent_limit_middleware` 限制同时运行的子 Agent 数量
- 子 Agent 超时机制作为最终兜底

---

## 问题 8：如何调整检测灵敏度？

```yaml
loop_detection:
  hash_window: 20        # 滑动窗口大小
  hash_warn_threshold: 3  # 哈希重复警告阈值
  hash_stop_threshold: 5  # 哈希重复强停阈值
  freq_warn_threshold: 30 # 频率警告阈值
  freq_stop_threshold: 50 # 频率强停阈值
```

如果 Agent 在做合理的反复尝试（如逐步调试），可适当放宽阈值。

---

## 数据流概览

```
Agent 调用工具 bash("npm test")
    │
    ▼ loop_detection_middleware（after_tool 钩子）
    │
    ▼ 更新线程调用历史
    │
    ▼ 哈希检测
    ├── 窗口内重复 < 3 → 正常
    ├── 窗口内重复 = 3 → 警告（追加提示）
    └── 窗口内重复 ≥ 5 → 强停（移除 tool_calls）
    │
    ▼ 频率检测
    ├── bash 调用 < 30 → 正常
    ├── bash 调用 = 30 → 警告
    └── bash 调用 ≥ 50 → 强停
```
