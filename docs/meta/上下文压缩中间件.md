# 上下文压缩中间件

**问题**: 对话太长，token 超限，Agent 无法继续工作。

---

## 问题 1：什么时候该压缩？

**每次 LLM 调用前**（`before_model` 钩子）检查 token 使用量。超过 `max_tokens` 阈值时触发。

```yaml
summarization:
  max_tokens: 100000
```

未超限 → 跳过，不做任何处理。

---

## 问题 2：哪些消息能压缩，哪些不能？

不能一刀切。三阶段筛选：

| 阶段 | 动作 | 为什么 |
|------|------|-------|
| 基础分区 | 系统提示 + 最近 N 条保留，中间的标为可压缩 | 粗切，可能误伤 |
| 技能救援 | 从可压缩区中捞出技能调用 | Agent 还在使用这些技能 |
| Reminder 保护 | 从可压缩区中捞出 todo_reminder | Agent 还有任务没完成 |

三阶段是递进的——每阶段都比上阶段更精细地保护重要信息。

---

## 问题 3：压缩后用什么替代？

对可压缩区的消息调用 LLM 生成一段摘要，用摘要 HumanMessage 替换原始消息。摘要包含关键决策、上下文结论、已解决的问题等。

---

## 问题 4：记忆系统怎么办？

消息被删除前，`memory_flush_hook` 被触发。它从即将删除的消息中提取用户偏好、关键事实等信息，**立即**排队给记忆系统（`add_nowait`，跳过 30 秒防抖）。

为什么不等防抖：消息即将永久删除，再等 30 秒就来不及了。

---

## 问题 5：记忆本身会膨胀吗？需要压缩吗？

记忆系统有独立的防膨胀机制，**不需要像对话那样压缩**：

| 防线 | 限制 | 怎么做的 |
|------|------|---------|
| Facts 数量上限 | 最多 100 条 | 超出后按**置信度降序**截断，低置信度的被丢弃 |
| 置信度门槛 | ≥ 0.7 才保留 | LLM 提取事实时打置信度分，低于门槛直接丢弃 |
| 注入 token 预算 | 最多 2000 token | 注入到对话时，从高置信度开始逐条添加，超出预算就停止 |

```
记忆 JSON（最多 100 条 facts）
    │
    ▼ 注入到对话时
format_memory_for_injection()
    ├── 按置信度排序 facts
    ├── 逐条添加，累计 token
    └── 超过 max_injection_tokens (2000) → 停止添加
    │
    ▼ 结果
Agent 看到的是记忆的"精华子集"，不是完整记忆
```

所以记忆的增长是被**上限 + 置信度筛选 + 注入预算**三重控制的。即使长期使用，记忆也不会导致上下文膨胀。

---

## 问题 6：用户能看到压缩吗？

能。被压缩的消息移到前端历史记录区域，用户向上滚动仍可查看。只是 Agent 的"工作记忆"里只保留摘要。

前端通过 SSE 事件 `SummarizationMiddleware.before_model` 感知压缩，将被移除的消息追加到历史。摘要消息标记 `name="summary"`，前端渲染时自动隐藏。

---

## 问题 7：和其他中间件的执行顺序？

```
DynamicContext（注入上下文）→ Summarization（压缩）→ Todo（注入提醒）→ ViewImage（注入图像）
```

Todo 在压缩**后**注入——确保 todo_reminder 不被压缩掉。

---

## 问题 8：如何调整压缩行为？

| 想法 | 配置 |
|------|------|
| 允许更长对话 | `max_tokens: 150000` |
| 保留更多近期消息 | `keep_recent_messages: 20` |
| 用更便宜的模型生成摘要 | `summary_model: "gpt-4o-mini"` |
| 保留更多技能上下文 | `preserve_recent_skill_count: 10` |
| 关闭压缩 | `enabled: false` |

---

## 问题 9：压缩后 Agent 忘记重要信息怎么办？

两个防线：
1. **三阶段保护**已尽量保留重要信息（技能、reminder、近期消息）
2. 如果仍不足，调大 `keep_recent_messages` 或 `max_tokens`

摘要本身也保留了关键决策和结论，不是完全遗忘。

---

## 数据流概览

```
120 条消息（10 万 token）
    │
    ▼ 基础分区
段 B = 第 1~38 轮（可压缩区）
    │
    ▼ 技能救援
救回 8 条技能相关消息
    │
    ▼ Reminder 保护
救回 2 条 todo_reminder
    │
    ▼ 生成摘要 + 记忆 flush
段 B → 1 条摘要 + 记忆已提取
    │
    ▼ 替换
41 条消息（4 万 token）→ Agent 继续工作
```
