# 技能加载链路

**问题**: Agent 需要领域专业知识和工作流指导，但把所有技能塞进系统提示会撑爆上下文窗口，需要按需、安全地加载。

---

## 问题 1：技能的格式是什么？

每个技能是一个目录，包含 `SKILL.md` 文件：

```markdown
---
name: react-development
description: "React 前端开发技能"
allowed-tools:
  - bash
  - write_file
  - read_file
  - web_search
---

## 技能详细指令
你是一个 React 开发专家，遵循以下规范：
- 使用 TypeScript
- 函数组件 + Hooks
...
```

YAML frontmatter 是元数据，正文是指令内容。

---

## 问题 2：技能怎么发现和加载？

从文件系统扫描技能目录：

```yaml
skills:
  container_path: /mnt/skills  # Agent 视角的路径
  path: skills                 # 物理路径
```

```
skills/
    ├── react-development/
    │   └── SKILL.md
    ├── python-data-science/
    │   └── SKILL.md
    └── devops/
        └── SKILL.md
```

`skill_storage` 扫描目录，`parser` 解析 SKILL.md，提取元数据和指令内容。

---

## 问题 3：所有技能都加载到上下文吗？

**不是**。技能经过筛选后才注入：

1. **启用/禁用过滤**：通过 `extensions_config.json` 控制
2. **相关性匹配**：只加载与当前任务相关的技能
3. **工具策略过滤**：技能声明的 `allowed-tools` 会限制 Agent 可用的工具集

```json
{
  "skills": {
    "react-development": {"enabled": true},
    "legacy-skill": {"enabled": false}
  }
}
```

---

## 问题 4：技能的工具策略怎么工作？

`tool_policy_mode` 控制工具过滤策略：

| 模式 | 行为 |
|------|------|
| `allowlist` | 只保留技能 `allowed-tools` 中声明的工具 |
| `none` | 不过滤，保留所有工具 |

```yaml
skills:
  tool_policy_mode: "allowlist"
```

当启用 `allowlist` 模式时，所有已加载技能的 `allowed-tools` 取并集，只有在此集合中的工具才能被 Agent 使用。这防止了 Agent 在特定技能场景下调用不相关的工具。

---

## 问题 5：技能有安全扫描吗？

有。`security_scanner` 在加载时检查 SKILL.md：

- 检测恶意指令（如试图修改沙箱配置）
- 检测危险工具声明
- 验证元数据格式

不通过的技能会被跳过并记录警告。

---

## 问题 6：技能怎么注入到 Agent？

`dynamic_context_middleware` 在每次 LLM 调用前，将激活技能的指令内容拼接到系统提示中：

```
原始系统提示
    │
    ▼ dynamic_context_middleware
拼接技能指令 + 记忆 + 其他上下文
    │
    ▼ 完整系统提示 → 发给 LLM
```

同时 `deferred_tool_filter_middleware` 根据技能的工具策略过滤可用工具。

---

## 问题 7：技能可以在运行时热加载吗？

可以。`skill_storage` 会在每次请求时检查技能目录的变化（通过文件修改时间）。新增或更新的技能会在下一次 LLM 调用时生效，无需重启 Agent。

---

## 问题 8：技能和 MCP 工具有什么关系？

技能可以声明 MCP 工具作为 `allowed-tools`，从而让 Agent 在特定技能场景下使用外部工具。但 MCP 工具的加载是独立的（由 MCP 缓存管理），技能只是控制"能不能用"。

---

## 数据流概览

```
skills/ 目录
    │
    ▼ skill_storage 扫描
    │
    ▼ parser 解析 SKILL.md
    │
    ▼ extensions_config 过滤（启用/禁用）
    │
    ▼ security_scanner 安全检查
    │
    ▼ 激活的技能列表
    ├── → dynamic_context_middleware（注入指令到系统提示）
    └── → deferred_tool_filter_middleware（过滤可用工具）
    │
    ▼ Agent 使用技能
```
