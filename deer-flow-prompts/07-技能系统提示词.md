```python
class SkillLoader:
    def __init__(self, public_dir: Path, custom_dir: Path):
        self.public_dir = public_dir
        self.custom_dir = custom_dir

    def load_all(self) -> dict[str, Skill]:
        skills = {}
        # 先加载 public（优先级低）
        for skill in self._load_from_dir(self.public_dir):
            skills[skill.name] = skill
        # 再加载 custom（优先级高，覆盖 public）
        for skill in self._load_from_dir(self.custom_dir):
            skills[skill.name] = skill
        return skills
```

## 🔍 设计思路解读重点

### 设计理念1：YAML Frontmatter

**为什么用 YAML Frontmatter？**
- 人类可读可写
- 支持丰富的元数据
- Markdown 内容与元数据分离

**设计权衡**：
- 优点：用户友好、易于编辑
- 缺点：需要解析器、格式错误风险
- 替代方案：JSON、TOML、独立元数据文件

### 设计理念2：双目录架构

**为什么分 public 和 custom？**
- public：官方维护的技能
- custom：用户自定义的技能
- custom 覆盖 public：支持技能定制

**设计挑战**：
- 技能名称冲突处理
- 优先级规则明确
- 增量加载性能

### 设计理念3：.skill 归档格式

**为什么设计 .skill 归档？**
- 技能分发
- 版本管理
- 安装便利

**安全考虑**：
- 禁止路径穿越
- 禁止符号链接
- 验证文件类型

### 设计理念4：渐进式加载

**为什么需要渐进式加载？**
- 按需加载技能
- 减少启动时间
- 动态发现新技能

**设计权衡**：
- 优点：性能优化
- 缺点：复杂度增加
- 实现方式：文件监听、mtime 检测
