## 🔍 设计思路解读重点

### 设计理念1：纯函数设计

**为什么工具函数设计为纯函数？**
- 无副作用，易于测试
- 可预测的输出
- 易于并发调用

**设计原则**：
```python
# ✅ 纯函数
def add(a: int, b: int) -> int:
    return a + b

# ❌ 有副作用
counter = 0
def add_with_side_effect(a: int, b: int) -> int:
    global counter
    counter += 1
    return a + b
```

### 设计理念2：单一职责

**为什么每个文件只做一件事？**
- 文件转换 → file_conversion.py
- 网络请求 → network.py
- 可读性 → readability.py

**设计权衡**：
- 优点：职责清晰、易于维护
- 缺点：文件数量多
- 组织原则：按功能域划分
