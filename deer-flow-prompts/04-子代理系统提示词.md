## 🔍 设计思路解读重点

### 设计理念1：双线程池架构

**为什么分离调度与执行？**
- **调度池**：单线程确保并发控制原子性
- **执行池**：多线程支持高并发子代理

**设计权衡**：
- 优点：职责分离、避免竞态、资源隔离
- 缺点：复杂度增加、上下文切换开销
- 替代方案：单线程池 + 队列、进程池

### 设计理念2：并发控制

**为什么限制为 3？**
- 资源约束：每个子代理需要独立 LLM 上下文
- 质量保证：少量并发便于监控
- 成本控制：限制并发量控制成本

**15分钟超时的考量**：
- 覆盖大多数复杂任务
- 避免僵尸任务永久占用资源
- 配合重试机制实现容错

### 设计理念3：状态继承与隔离

**为什么继承父代理状态？**
- 子代理需要访问对话上下文
- 避免重复传递配置
- 保持任务连贯性

**为什么需要覆盖某些状态？**
- agent_name：区分父/子代理
- instructions：子代理有独立指令
- tools：子代理只能访问被允许的工具

### 设计理念4：工具过滤

**为什么需要工具过滤？**
- 最小权限原则
- 安全性：子代理不能执行危险操作
- 灵活性：不同子代理不同工具集

**过滤逻辑**：
```python
# 优先级：disallowed > allowed > parent_tools
1. 应用黑名单移除危险工具
2. 应用白名单只保留允许的工具
3. 都为空则保留全部父代理工具
```

### 设计理念5：后台任务模式

**为什么支持后台执行？**
- 长时间任务不阻塞主代理
- 异步工作流支持
- 进度查询能力

**状态转换**：
```
spawn → PENDING → RUNNING → COMPLETED/FAILED
         │          │          │
         │          │          └─→ get_result()
         │          └─→ poll()
         └─→ cancel()
```
