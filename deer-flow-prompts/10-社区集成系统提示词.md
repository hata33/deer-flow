## 🔍 设计思路解读重点

### 设计理念1：统一工具接口

**为什么所有集成都提供 Tool 接口？**
- Agent 无需关心底层实现
- 统一的调用方式
- 易于添加新集成

**接口规范**：
```python
Tool(
    name="service_name_action",
    description="What this tool does",
    func=async_function,
    parameters={...}
)
```

### 设计理念2：独立包结构

**为什么每个服务独立目录？**
- 模块边界清晰
- 可选依赖管理
- 独立测试

**设计权衡**：
- 优点：解耦、可维护
- 缺点：代码重复风险
- 缓解：共享基础库

### 设计理念3：客户端封装

**为什么需要客户端类？**
- 封装 API 调用细节
- 处理认证和重试
- 统一错误处理

**设计模式**：
```python
class ServiceClient:
    def __init__(self, api_key, base_url):
        self.api_key = api_key
        self.base_url = base_url

    async def _call(self, endpoint, params):
        # 统一的 API 调用逻辑
        # 错误处理
        # 重试逻辑
```

### 设计理念4：可插拔架构

**为什么设计为可插拔？**
- 用户可以选择需要的集成
- 减少依赖和启动时间
- 支持自定义集成

**实现方式**：
- 动态导入
- 配置驱动
- 依赖注入
