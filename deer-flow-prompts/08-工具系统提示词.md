            result = await tool.func(**arguments)
        else:

## 🔍 设计思路解读重点

### 设计理念1：统一工具接口

**为什么需要统一接口？**
- Agent 无需关心工具实现细节
- 易于添加新工具
- 统一的错误处理

**设计权衡**：
- 优点：一致性、可扩展
- 缺点：抽象层开销
- 接口设计：name, description, parameters, func

### 设计理念2：工具注册表模式

**为什么用注册表？**
- 集中管理工具
- 动态发现工具
- 支持工具热加载

**设计挑战**：
- 名称冲突处理
- 循环依赖检测
- 工具版本管理

### 设计理念3：内置工具集

**为什么提供内置工具？**
- 核心功能开箱即用
- 保证系统可用性
- 作为工具开发示例

**工具分类**：
- 交互类：clarification, present_file
- 管理类：task, setup_agent
- 信息类：tool_search, view_image

### 设计理念4：工具搜索

**为什么需要工具搜索？**
- 工具数量增长后的发现需求
- 帮助 Agent 找到合适工具
- 语义搜索 vs 关键词搜索

**设计挑战**：
- 搜索相关性
- 性能优化
- 结果排序
