```python
class StreamBridge(ABC):
    @abstractmethod
    async def publish(self, thread_id: str, chunk: Chunk):
        # 发布 chunk
        pass

    @abstractmethod
    async def subscribe(self, thread_id: str) -> AsyncIterator[Chunk]:
        # 订阅 chunk 流
        pass

class MemoryStreamBridge(StreamBridge):
    def __init__(self):
        self.queues: dict[str, Queue] = {}

    async def publish(self, thread_id, chunk):
        queue = self.queues.get(thread_id)
        if queue:
            await queue.put(chunk)
```

## 🔍 设计思路解读重点

### 设计理念1：Provider 模式

**为什么运行时也用 Provider？**
- 支持多种存储后端
- 可测试性（内存存储）
- 部署灵活性

**Provider 类型**：
- StoreProvider: SQLite, PostgreSQL, Redis
- StreamBridge: Memory, WebSocket, SSE

### 设计理念2：异步优先

**为什么全异步设计？**
- 高并发处理
- 非阻塞 I/O
- 流式输出的自然支持

**设计挑战**：
- 异步上下文管理
- 并发控制
- 错误传播

### 设计理念3：流式桥接

**为什么需要流式桥接？**
- 解耦执行与传输
- 支持多种传输协议
- 背压处理

**桥接模式**：
```
LangGraph Stream → StreamBridge → WebSocket/SSE/HTTP
```

### 设计理念4：状态持久化

**为什么需要持久化？**
- 长时间对话
- 断点恢复
- 状态共享

**存储策略**：
- 每次调用后保存
- 检查点机制
- 增量更新
