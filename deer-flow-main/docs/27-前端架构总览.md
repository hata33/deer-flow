# 【27】前端架构总览

## 1. 模块全局定位

- **所属项目**：deer-flow
- **层级位置**：`frontend/`
- **核心作用**：提供Next.js 16 + React 19 Web用户界面，实现对话交互、技能管理、设置配置
- **业务价值**：作为系统的"用户交互层"，暴露AI代理能力给最终用户，提供直观的对话、文件上传、技能管理体验
- **设计初衷**：设计用于解决"AI应用可用性"问题——通过现代Web界面降低AI代理使用门槛，提供可视化操作与实时反馈

## 2. 技术栈与架构设计

**核心技术栈**：
- **框架**：Next.js 16（App Router）
- **UI库**：React 19 + TypeScript 5.8
- **样式**：Tailwind CSS 4
- **状态管理**：TanStack Query（服务端状态）+ localStorage（用户设置）
- **组件库**：Shadcn UI + Vercel AI SDK

**架构模式**：
- **Server Components优先**：默认使用服务端组件，按需添加`"use client"`
- **流式响应**：通过LangGraph SDK的SSE流式传输AI响应
- **状态分层**：服务端状态（TanStack Query）与客户端状态（React Context）分离

## 4. 核心模块解析

### 4.1 线程管理（core/threads/）

**核心功能**：
- `useThreadStream`：处理流式AI响应
- `useSubmitThread`：提交用户消息
- `useThreads`：管理线程列表

**设计亮点**：
- **流式事件处理**：解析LangGraph SSE事件（`messages-tuple`、`values`、`end`）
- **乐观更新**：立即显示用户消息，AI响应流式追加
- **错误处理**：网络错误、API错误的用户友好提示

### 4.2 API客户端（core/api/）

**核心功能**：
- `getAPIClient()`：获取LangGraph SDK客户端单例
- 环境变量配置：`NEXT_PUBLIC_LANGGRAPH_BASE_URL`

**设计亮点**：
- **单例模式**：全局共享客户端，避免重复创建
- **nginx代理**：生产环境通过nginx统一入口，开发环境直连后端

### 4.3 技能系统（core/skills/）

**核心功能**：
- `useSkills`：技能列表查询
- `useInstallSkill`：技能安装
- 技能详情、启用/禁用管理

**设计亮点**：
- **TanStack Query缓存**：自动缓存技能列表，减少API调用
- **乐观更新**：安装后立即刷新列表，提升响应速度

### 4.4 MCP集成（core/mcp/）

**核心功能**：
- `useMCPConfig`：MCP服务器配置查询
- `useEnableMCPServer`：服务器启用/禁用

**设计亮点**：
- **配置热更新**：修改后立即生效，无需重启
- **类型安全**：TypeScript接口定义配置结构

### 4.5 工件系统（core/artifacts/）

**核心功能**：
- 工件列表查询
- 工件内容获取
- 工件预览（HTML/PDF/图片）

**设计亮点**：
- **虚拟路径映射**：`/threads/{id}/artifacts/{path}`映射到实际文件
- **安全下载**：强制下载HTML/SVG等活跃内容，防止XSS

## 5. 状态管理策略

### 5.1 服务端状态（TanStack Query）

**适用场景**：
- 线程列表、技能列表、模型列表
- MCP配置、记忆状态
- 工件列表、上传文件列表

**优势**：
- 自动缓存与重新验证
- 后台数据同步
- 乐观更新支持

### 5.2 客户端状态（localStorage）

**适用场景**：
- 用户设置（模型选择、主题）
- UI状态（侧边栏展开、面板显示）

**优势**：
- 跨会话持久化
- 无需服务器存储
- 快速访问

## 6. 路由设计

### 6.1 页面路由

- `/`：首页（项目介绍、功能展示）
- `/workspace/chats/[thread_id]`：对话页面

### 6.2 API路由

通过nginx代理：
- `/api/langgraph/*` → LangGraph Server（2024）
- `/api/*` → Gateway API（8001）

## 7. 组件设计模式

### 7.1 Server Components优先

**原则**：默认使用服务端组件，只在需要交互时添加`"use client"`

**优势**：
- 减少客户端JavaScript体积
- SEO友好
- 更快的首屏加载

### 7.2 组件层次结构

```
page.tsx (Server Component)
├── Layout (Server Component)
│   ├── Header (Client Component)
│   └── Sidebar (Client Component)
└── ThreadView (Client Component)
    ├── MessageList (Client Component)
    ├── InputArea (Client Component)
    └── ArtifactPanel (Client Component)
```
