deer-flow-main\backend\app\gateway\app.py + deps.py

整个 agent 调用流程的前提条件——app 启动时初始化全局单例，后续路由层和 service 层从 app.state 上取用

---

**app 启动入口**

``` python
@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncGenerator[None, None]:
    # 1. 加载配置（触发 config.yaml 读取，详见 005.1）
    get_app_config()

    # 2. 初始化 LangGraph 运行时组件
    async with langgraph_runtime(app):

        # 3. 启动 IM 频道服务（可选）
        await start_channel_service()

        yield  # app 运行中

        # 关闭时停止频道
        await stop_channel_service()
```

`lifespan` 是 FastAPI 的生命周期管理器，app 启动时执行 yield 前的逻辑，关闭时执行 yield 后的逻辑。`langgraph_runtime(app)` 在这里作为上下文管理器使用

**全局单例创建（deps.py langgraph_runtime）**

``` python
@asynccontextmanager
async def langgraph_runtime(app: FastAPI) -> AsyncGenerator[None, None]:
    from deerflow.agents.checkpointer.async_provider import make_checkpointer
    from deerflow.runtime import make_store, make_stream_bridge

    async with AsyncExitStack() as stack:
        app.state.stream_bridge = await stack.enter_async_context(make_stream_bridge())
        app.state.checkpointer = await stack.enter_async_context(make_checkpointer())
        app.state.store = await stack.enter_async_context(make_store())
        app.state.run_manager = RunManager()
        yield
```

四个全局单例：

| 单例 | 创建方式 | 用途 |
|------|----------|------|
| `stream_bridge` | `make_stream_bridge()` | StreamBridge 异步队列，agent 推流 + sse_consumer 消费 |
| `checkpointer` | `make_checkpointer()` | 状态持久化，保存/恢复 LangGraph 图状态 |
| `store` | `make_store()` | 线程元数据存储，支持 /threads/search |
| `run_manager` | `RunManager()` | 运行记录管理，状态追踪、取消、并发策略 |

`AsyncExitStack` 确保所有带 `__aenter__/__aexit__` 的资源在 app 关闭时正确释放。`stream_bridge`、`checkpointer`、`store` 都是需要清理的资源（队列、数据库连接等），`RunManager` 是纯内存对象不需要

**全局单例获取（deps.py getters）**

``` python
def get_stream_bridge(request: Request) -> StreamBridge:
    bridge = getattr(request.app.state, "stream_bridge", None)
    if bridge is None:
        raise HTTPException(status_code=503, detail="Stream bridge not available")
    return bridge
```

四个 getter 函数（`get_stream_bridge`、`get_run_manager`、`get_checkpointer`、`get_store`）从 `request.app.state` 上读取。缺失时返回 503（`get_store` 例外，允许返回 `None`）

路由层和 service 层通过这些 getter 获取单例，不直接操作 `app.state`。这样保证了：
- 单例在 app 启动时统一创建
- 各模块通过 request 获取，不持有引用
- 生命周期由 `AsyncExitStack` 管理

**在调用链路中的位置**

```
app.py lifespan()
  ├─ get_app_config()                    加载配置（005.1）
  ├─ langgraph_runtime(app)              创建四个全局单例
  │   ├─ app.state.stream_bridge
  │   ├─ app.state.checkpointer
  │   ├─ app.state.store
  │   └─ app.state.run_manager
  └─ start_channel_service()             启动 IM 频道（可选）

...后续每次请求...

路由层 thread_runs.py
  ├─ get_stream_bridge(request)          从 app.state 取 bridge
  ├─ get_run_manager(request)            从 app.state 取 run_manager
  └─ start_run() 内部
      ├─ get_checkpointer(request)       从 app.state 取 checkpointer
      └─ get_store(request)              从 app.state 取 store
```

> 本步骤：app 启动时通过 `langgraph_runtime()` 在 `app.state` 上创建 bridge、checkpointer、store、run_manager 四个全局单例。后续路由层和 service 层通过 deps.py 的 getter 函数按需获取，不持有引用，生命周期由 AsyncExitStack 管理
