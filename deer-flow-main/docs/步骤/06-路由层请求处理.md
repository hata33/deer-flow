deer-flow-main\backend\app\gateway\routers\thread_runs.py
文件承接 `/{thread_id}/runs/stream` 的路由请求

`stream_run(thread_id, body: RunCreateRequest, request: Request)`

接收参数：`thread_id` 从路径获取，`body` 为请求传参 JSON，`request` 为 FastAPI 全局应用实例

``` python
bridge = get_stream_bridge(request)
run_mgr = get_run_manager(request)
record = await start_run(body, thread_id, request)
```

从 `app.state` 获取全局的 `stream_bridge`（流式桥）和 `run_manager`（运行管理器），然后调用 `start_run` 创建 `record`

最后执行 `StreamingResponse` 响应，传入 `sse_consumer(bridge, record, request, run_mgr)` 向前端推流

---

`record` 在 `deer-flow-main\backend\app\gateway\services.py` 的 `start_run` 中创建

``` python
bridge = get_stream_bridge(request)
run_mgr = get_run_manager(request)
checkpointer = get_checkpointer(request)
store = get_store(request)
```

获取 `app.state` 上的四个全局单例：`bridge`、`run_mgr`、`checkpointer`、`store`

``` python
disconnect = DisconnectMode.cancel if body.on_disconnect == "cancel" else DisconnectMode.continue_
```

`body.on_disconnect` 控制客户端断开连接时的行为：

- `"cancel"` → 设置 `DisconnectMode.cancel`，流式运行立即取消
- 其他值 → 设置 `DisconnectMode.continue_`，断开后运行继续在后台执行

``` python
try:
    record = await run_mgr.create_or_reject(
        thread_id,
        body.assistant_id,
        on_disconnect=disconnect,
        metadata=body.metadata or {},
        kwargs={"input": body.input, "config": body.config},
        multitask_strategy=body.multitask_strategy,
    )
except ConflictError as exc:
    raise HTTPException(status_code=409, detail=str(exc)) from exc
except UnsupportedStrategyError as exc:
    raise HTTPException(status_code=501, detail=str(exc)) from exc
```

创建 `record`，包含冲突检测（409）和不支持策略检测（501）的错误捕获与抛出

``` python
store = get_store(request)
if store is not None:
    await _upsert_thread_in_store(store, thread_id, body.metadata)
```

确保 thread 在 store 中存在（对未通过 `POST /threads` 显式创建的无状态 run 也能在 `/threads/search` 中查到）

``` python
agent_factory = resolve_agent_factory(body.assistant_id)
graph_input = normalize_input(body.input)
config = build_run_config(thread_id, body.config, body.metadata)
stream_modes = normalize_stream_modes(body.stream_mode)
```

根据 `body` 传参准备运行所需各部分：

- `agent_factory`：解析 `assistant_id`，实际固定走 `make_lead_agent`
- `graph_input`：将 LangGraph Platform 输入格式转为 LangChain state dict
- `config`：组装 `RunnableConfig`（含 `thread_id`、`recursion_limit` 等）
- `stream_modes`：规范化流式模式参数（默认 `["values"]`）

``` python
task = asyncio.create_task(
    run_agent(
        bridge, run_mgr, record,
        checkpointer=checkpointer,
        store=store,
        agent_factory=agent_factory,
        graph_input=graph_input,
        config=config,
        stream_modes=stream_modes,
        stream_subgraphs=body.stream_subgraphs,
        interrupt_before=body.interrupt_before,
        interrupt_after=body.interrupt_after,
    )
)
record.task = task
```

组装后台任务，核心是 `run_agent`，由 `asyncio.create_task` 异步调度

``` python
if store is not None:
    asyncio.create_task(_sync_thread_title_after_run(task, thread_id, checkpointer, store))
```

运行完成后，将 `TitleMiddleware` 从检查点生成的标题同步到 Store 记录中，使 `/threads/search` 返回正确的标题

``` python
return record
```

返回 `record`，后续由 `sse_consumer(bridge, record, request, run_mgr)` 消费推流

---

全局单例的获取来自 `deer-flow-main\backend\app\gateway\deps.py`，从 `request.app.state` 上读取，缺失时返回 503（`store` 例外，允许为 `None`）。初始化在 `app.py` 中通过 `langgraph_runtime` 上下文管理器完成

> 本步骤：接收前端流式请求 → 创建 run 记录 → 在后台异步启动 agent 执行 → 通过 SSE 向前端实时推流
