**执行流程**

```
agent.astream() 驱动 LangGraph 图执行
  → ReAct 循环：LLM 调用 → 工具执行 → LLM 调用 → ...
  → 每步产出 chunk

  → serialize() 序列化为 JSON

```

```
StreamBridge 队列（生产消费解耦）
  → sse_consumer（004）异步迭代 bridge.subscribe() 取出事件
  → format_sse() 格式化为 SSE 帧（event: + data: + 空行）
  → StreamingResponse 通过 HTTP 长连接推送给前端
  → 前端 useStream hook 解析 SSE 帧更新 UI
```

---

**设计要点**

**生产消费解耦**：`bridge.publish()` 和 `bridge.subscribe()` 通过异步队列解耦。agent 执行（生产）和 SSE 推流（消费）在不同协程中运行，互不阻塞

**单次执行全程链路**：从 body 参数进入到前端收到 SSE 流，整个链路在同一个请求的生命周期内完成。`start_run` 创建 task 并立即返回 record，`stream_run` 用 record 构建 StreamingResponse，后台 task 的产出通过 bridge 传递到 sse_consumer 再到前端

**参数来源清晰**：用户问题来自 body，agent 行为来自 body + config.yaml 的合并，基础设施（bridge、checkpointer、store）来自 app.state 全局单例

> 本步骤：`agent.astream(graph_input, config, stream_mode)` 是整个系统的分界点，用户问题 + 配置 + 流式模式 → agent ReAct 循环执行 → chunk 序列化 → bridge 推入队列 → sse_consumer 消费 → SSE 帧推送给前端
