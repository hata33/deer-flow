工具加载——四层工具源与延迟加载机制

---

## 一、入口函数

**文件**：`tools/tools.py` → `get_available_tools(groups, include_mcp, model_name, subagent_enabled)`

每次 `make_lead_agent()` 调用时执行，返回 `list[BaseTool]`

---

## 二、四层工具源

```
get_available_tools()
  │
  ├─ 第一层：配置工具（config.yaml → resolve_variable 反射加载）
  ├─ 第二层：内置工具（固定列表 + 条件注入）
  ├─ 第三层：MCP 工具（缓存读取 + 可选延迟注册）
  └─ 第四层：ACP 工具（配置驱动）
```

### 第一层：配置工具

从 `config.yaml` 的 `tools` 列表加载，通过 `resolve_variable("module.path:var")` 反射创建工具实例

```yaml
# config.yaml 示例
tools:
  - name: web_search
    use: "langchain_community.tools:DDGSearchResults"
    group: search
  - name: read_file
    use: "deerflow.sandbox.tools:read_file_tool"
    group: fs
```

过滤规则：
- `groups` 参数不为 None 时，只加载匹配 `tool.group` 的工具
- 使用 LocalSandboxProvider 时，自动移除宿主机 bash 工具（安全过滤）

### 第二层：内置工具

```python
BUILTIN_TOOLS = [
    present_file_tool,       # 向用户展示输出文件
    ask_clarification_tool,  # 请求用户澄清（被 ClarificationMiddleware 拦截）
]
```

条件注入的内置工具：

| 工具 | 注入条件 | 用途 |
|------|----------|------|
| `task_tool` | `subagent_enabled=True` | 启动子代理并行执行 |
| `view_image_tool` | `model.supports_vision=True` | 查看图片内容 |

### 第三层：MCP 工具

从 MCP Server 缓存中读取，不直接连接 MCP Server（启动时预加载）：

```python
from deerflow.mcp.cache import get_cached_mcp_tools
mcp_tools = get_cached_mcp_tools()
```

**延迟加载模式（tool_search）**：

当 `config.tool_search.enabled=True` 时，MCP 工具的处理方式改变：

```
正常模式：MCP 工具直接加入工具列表 → LLM 看到所有工具 schema
延迟模式：MCP 工具注册到 DeferredToolRegistry → 只暴露 tool_search 工具
         → LLM 需要时调用 tool_search(name) → 运行时动态加载 schema
```

延迟模式的好处：避免大量 MCP 工具 schema 占用 context window

相关文件：`tools/builtins/tool_search.py`

### 第四层：ACP 工具

ACP（Agent Communication Protocol）代理工具，允许调用外部代理（如 Claude Code、Codex）：

```python
from deerflow.config.acp_config import get_acp_agents
from deerflow.tools.builtins.invoke_acp_agent_tool import build_invoke_acp_agent_tool

acp_agents = get_acp_agents()
if acp_agents:
    acp_tools.append(build_invoke_acp_agent_tool(acp_agents))
```

ACP 代理运行在独立工作空间（`/mnt/acp-workspace/`），与主 agent 工作空间隔离

## 三、工具合并与去重

四层工具按顺序合并：`loaded_tools + builtin_tools + mcp_tools + acp_tools`

去重规则：用户在 config.yaml 中配置的工具优先。SDK 层 `create_deerflow_agent()` 在合并时按工具名去重：

```python
existing_names = {t.name for t in effective_tools}
for t in extra_tools:
    if t.name not in existing_names:
        effective_tools.append(t)
```
