## 一、架构总览

```
对话完成
  │
  ├─ MemoryMiddleware.after_agent()
  │   └─ 过滤消息 → 排队
  │
  ├─ MemoryUpdateQueue（防抖队列）
  │   └─ debounce 计时器 → 批量处理
  │
  └─ MemoryUpdater.update_memory()
      ├─ 加载 memory.json
      ├─ LLM 分析对话 → JSON 更新指令
      ├─ 应用更新（sections + facts）
      └─ 原子写入文件

下轮对话构建时
  └─ _get_memory_context() → 读取 memory.json → 注入 <memory> 段落
```

## 二、存储层

**文件**：`agents/memory/storage.py`

### 2.1 memory.json 结构

```json
{
  "version": "1.0",
  "lastUpdated": "2026-05-15T08:00:00Z",
  "user": {
    "workContext":    {"summary": "核心贡献者，项目 A...", "updatedAt": "..."},
    "personalContext": {"summary": "中英双语，偏好...", "updatedAt": "..."},
    "topOfMind":      {"summary": "多个并行关注点...", "updatedAt": "..."}
  },
  "history": {
    "recentMonths":      {"summary": "近期活动详细摘要...", "updatedAt": "..."},
    "earlierContext":    {"summary": "历史模式...", "updatedAt": "..."},
    "longTermBackground": {"summary": "持久背景...", "updatedAt": "..."}
  },
  "facts": [
    {
      "id": "fact_a1b2c3d4",
      "content": "用户偏好 TypeScript",
      "category": "preference",
      "confidence": 0.9,
      "createdAt": "2026-05-15T08:00:00Z",
      "source": "thread_abc123"
    }
  ]
}
```

**各 section 说明**：

| Section | 长度 | 更新频率 | 内容 |
|---------|------|----------|------|
| `user.workContext` | 2-3 句 | 中等 | 职业、公司、主要技术栈 |
| `user.personalContext` | 1-2 句 | 低 | 语言、沟通偏好、兴趣 |
| `user.topOfMind` | 3-5 句 | 高 | 当前多个并行关注点 |
| `history.recentMonths` | 4-6 句/1-2 段 | 中等 | 最近 1-3 月活动 |
| `history.earlierContext` | 3-5 句 | 低 | 3-12 月前的模式 |
| `history.longTermBackground` | 2-4 句 | 极低 | 不变的背景信息 |
| `facts` | 列表 | 高 | 离散事实，按置信度排序 |

**Facts 类别**：`preference`（偏好）、`knowledge`（知识）、`context`（背景）、`behavior`（行为）、`goal`（目标）

## 三、更新队列（防抖机制）

**文件**：`agents/memory/queue.py` → `MemoryUpdateQueue`

### 3.1 工作流程

```
MemoryMiddleware.after_agent()
  └─ queue.add(thread_id, messages, agent_name)
      ├─ 同 thread 已有待处理更新 → 替换为最新（去重）
      └─ 重置 debounce 计时器（默认 30s）

计时器到期
  └─ _process_queue()
      ├─ 取出所有待处理上下文
      ├─ 对每个上下文调用 MemoryUpdater.update_memory()
      └─ 更新间 sleep 0.5s 避免速率限制
```

### 3.2 防抖参数

```yaml
# memory_config
debounce_seconds: 30    # 每次新消息重置计时器，30s 无新消息才触发
```

目的：用户快速连续对话时，等对话稳定后再批量更新记忆，避免每句话触发一次 LLM 调用

## 四、更新器（LLM 驱动）

**文件**：`agents/memory/updater.py` → `MemoryUpdater`

### 4.1 完整更新流程

```python
def update_memory(self, messages, thread_id, agent_name):
    # 1. 加载当前记忆
    current_memory = get_memory_data(agent_name)

    # 2. 格式化对话（过滤、截断）
    conversation_text = format_conversation_for_update(messages)

    # 3. LLM 调用
    prompt = MEMORY_UPDATE_PROMPT.format(
        current_memory=json.dumps(current_memory, indent=2),
        conversation=conversation_text,
    )
    response = model.invoke(prompt)

    # 4. 解析 JSON 响应
    update_data = json.loads(response_text)

    # 5. 应用更新
    updated_memory = self._apply_updates(current_memory, update_data, thread_id)

    # 6. 清除上传事件引用
    updated_memory = _strip_upload_mentions_from_memory(updated_memory)

    # 7. 原子写入
    return get_memory_storage().save(updated_memory, agent_name)
```

### 4.2 消息过滤

`_filter_messages_for_memory(messages)` 只保留：
- **Human 消息**：去除 `<uploaded_files>` 块（会话级数据不持久化）
- **无 tool_calls 的 AI 消息**：最终响应，不含中间工具调用

去除的理由：
- 工具调用是执行细节，不是用户信息
- 上传文件路径在下次会话不存在

### 4.3 LLM 提示词（MEMORY_UPDATE_PROMPT）

提示词要求 LLM 返回结构化 JSON：

```json
{
  "user": {
    "workContext":    {"summary": "...", "shouldUpdate": true/false},
    "personalContext": {"summary": "...", "shouldUpdate": true/false},
    "topOfMind":      {"summary": "...", "shouldUpdate": true/false}
  },
  "history": {
    "recentMonths":      {"summary": "...", "shouldUpdate": true/false},
    "earlierContext":    {"summary": "...", "shouldUpdate": true/false},
    "longTermBackground": {"summary": "...", "shouldUpdate": true/false}
  },
  "newFacts": [
    {"content": "...", "category": "preference|knowledge|context|behavior|goal", "confidence": 0.0-1.0}
  ],
  "factsToRemove": ["fact_id_1", "fact_id_2"]
}
```

### 4.4 更新应用规则

```python
def _apply_updates(self, current_memory, update_data, thread_id):
    # User sections：只在 shouldUpdate=True 时更新
    for section in ["workContext", "personalContext", "topOfMind"]:
        if section_data.get("shouldUpdate") and section_data.get("summary"):
            current_memory["user"][section] = {"summary": summary, "updatedAt": now}

    # History sections：同上
    for section in ["recentMonths", "earlierContext", "longTermBackground"]:
        ...

    # 删除 facts
    facts_to_remove = set(update_data.get("factsToRemove", []))
    current_memory["facts"] = [f for f in facts if f["id"] not in facts_to_remove]

    # 新增 facts（去重 + 置信度阈值过滤）
    for fact in new_facts:
        if confidence >= config.fact_confidence_threshold:
            if content not in existing_fact_keys:  # 按内容去重
                current_memory["facts"].append(fact_entry)

    # Facts 上限
    if len(current_memory["facts"]) > config.max_facts:
        current_memory["facts"] = sorted(
            current_memory["facts"],
            key=lambda f: f["confidence"],
            reverse=True
        )[:config.max_facts]

    return current_memory
```

### 4.5 上传事件清理

`_strip_upload_mentions_from_memory()` 从所有摘要和事实中移除文件上传相关句子。上传文件是会话级的，下次会话不存在，持久化到长期记忆会导致 agent 搜索不存在的文件

## 五、注入时机

记忆注入发生在 `apply_prompt_template()` → `_get_memory_context()` 中：

```
make_lead_agent(config)
  └─ apply_prompt_template()
      └─ _get_memory_context(agent_name)
          ├─ get_memory_data(agent_name)  → 从文件加载
          └─ format_memory_for_injection() → 格式化并截断
```

**这意味着**：
- 记忆注入是**构建时**行为，不是运行时行为
- 本轮对话产生的记忆更新（MemoryMiddleware 排队 → 防抖 → LLM 更新 → 写文件），本轮**看不到**
- **下一轮** `make_lead_agent()` 构建新 agent 时，才能读到上轮更新的记忆

### 时序图

```
第 N 轮对话
  ├─ make_lead_agent() → _get_memory_context() → 读取 memory.json（旧记忆）
  ├─ agent 执行（用户看到旧记忆驱动的回答）
  ├─ MemoryMiddleware.after_agent() → 排队更新
  └─ 防抖 30s → MemoryUpdater → 写入 memory.json（新记忆）

第 N+1 轮对话
  ├─ make_lead_agent() → _get_memory_context() → 读取 memory.json（新记忆）
  └─ agent 执行（用户看到新记忆驱动的回答）
```

## 六、配置

```yaml
# config.yaml → memory 段
memory:
  enabled: true
  injection_enabled: true
  storage_path: "memory.json"           # 文件路径
  storage_class: "deerflow.agents.memory.storage.FileMemoryStorage"
  debounce_seconds: 30                   # 防抖秒数
  max_injection_tokens: 2000             # 注入 token 预算
  fact_confidence_threshold: 0.5         # fact 置信度阈值
  max_facts: 100                         # facts 上限
  model_name: null                       # 记忆更新用的模型（null = 默认模型）
```

> 本文档：记忆系统由三层组成——FileMemoryStorage（JSON 文件 + mtime 缓存 + 原子写入）、MemoryUpdateQueue（线程安全的防抖队列）、MemoryUpdater（LLM 分析对话 + 结构化更新）。消息过滤只保留 human/最终 ai、去除上传事件。注入在构建时发生，更新有 30s 防抖延迟，下一轮才能读到新记忆
