工厂函数与模型解析——`make_lead_agent()` 五步组装与三级模型优先级

---

## 一、工厂函数入口

**文件**：`agents/lead_agent/agent.py` → `make_lead_agent(config: RunnableConfig)`

每次对话请求都会调用一次 `make_lead_agent()`，从运行时 config 中提取参数，动态组装 agent。不是预创建、不是单例——每次 `astream()` 前都重新构建

### 五步组装

```python
def make_lead_agent(config: RunnableConfig):
    cfg = config.get("configurable", {})

    # 第一步：提取运行时参数
    thinking_enabled    = cfg.get("thinking_enabled", True)
    reasoning_effort    = cfg.get("reasoning_effort", None)
    requested_model_name = cfg.get("model_name") or cfg.get("model")
    is_plan_mode        = cfg.get("is_plan_mode", False)
    subagent_enabled    = cfg.get("subagent_enabled", False)
    max_concurrent_subagents = cfg.get("max_concurrent_subagents", 3)
    is_bootstrap        = cfg.get("is_bootstrap", False)
    agent_name          = cfg.get("agent_name")

    # 第二步：解析模型名称（三级优先级）
    agent_config = load_agent_config(agent_name) if not is_bootstrap else None
    agent_model_name = agent_config.model if agent_config and agent_config.model else _resolve_model_name()
    model_name = requested_model_name or agent_model_name

    # 第三步：验证模型能力
    model_config = app_config.get_model_config(model_name)
    if thinking_enabled and not model_config.supports_thinking:
        thinking_enabled = False  # 降级

    # 第四步：注入 LangSmith 追踪元数据
    config["metadata"].update({...})

    # 第五步：调用 create_agent()
    return create_agent(
        model=create_chat_model(name=model_name, thinking_enabled=thinking_enabled),
        tools=get_available_tools(model_name=model_name, ...),
        middleware=_build_middlewares(config, model_name=model_name),
        system_prompt=apply_prompt_template(...),
        state_schema=ThreadState,
    )
```

### 两个分支

| 分支 | 条件 | 区别 |
|------|------|------|
| **引导模式** | `is_bootstrap=True` | 使用精简提示词 + `setup_agent` 工具，用于自定义智能体创建 |
| **默认模式** | 否 | 标准主智能体，支持自定义智能体名称、工具组、推理力度 |

引导模式的提示词只注入 `bootstrap` 技能，工具列表额外包含 `setup_agent`

## 二、模型三级优先级解析

**文件**：`agents/lead_agent/agent.py` → `_resolve_model_name()`

```
config.configurable.model_name / model     ← 优先级 1：请求参数
                    ↓ 未指定
agents_config[agent_name].model            ← 优先级 2：智能体配置
                    ↓ 未配置
config.yaml → models[0].name               ← 优先级 3：全局默认
```

### 源码逻辑

```python
def _resolve_model_name(requested_model_name=None):
    app_config = get_app_config()
    default_model_name = app_config.models[0].name if app_config.models else None

    if requested_model_name and app_config.get_model_config(requested_model_name):
        return requested_model_name          # 优先级 1

    if requested_model_name and requested_model_name != default_model_name:
        logger.warning(f"Model '{requested_model_name}' not found; fallback to default")
    return default_model_name                # 优先级 3
```

智能体配置（优先级 2）在 `make_lead_agent()` 中直接处理：

```python
agent_config = load_agent_config(agent_name)
agent_model_name = agent_config.model if agent_config and agent_config.model else _resolve_model_name()
model_name = requested_model_name or agent_model_name
```

### 校验链

```
model_name → app_config.get_model_config(model_name)
  ├─ None → 抛出 ValueError("No chat model could be resolved")
  ├─ supports_thinking=False + thinking_enabled=True → 降级为 thinking_enabled=False
  └─ 通过 → 创建模型实例
```

## 三、模型创建

**函数**：`create_chat_model(name, thinking_enabled, reasoning_effort)`

内部流程：
1. 从 `get_app_config().get_model_config(name)` 获取模型配置
2. 按 `provider` 字段反射创建 LLM 实例（如 `langchain_openai.ChatOpenAI`、`langchain_anthropic.ChatAnthropic`）
3. 如果 `thinking_enabled=True`，配置 thinking 参数
4. 如果 `reasoning_effort` 不为 None，设置推理力度（low/medium/high）

---

## 四、config.configurable 完整参数表

| 参数 | 类型 | 默认值 | 用途 |
|------|------|--------|------|
| `thinking_enabled` | bool | True | 是否启用思考模式 |
| `reasoning_effort` | str | None | 推理力度 "low"/"medium"/"high" |
| `model_name` / `model` | str | None | 指定模型名称 |
| `is_plan_mode` | bool | False | 是否启用计划模式（TodoList） |
| `subagent_enabled` | bool | False | 是否启用子代理 |
| `max_concurrent_subagents` | int | 3 | 最大并发子代理数 |
| `is_bootstrap` | bool | False | 引导模式 |
| `agent_name` | str | None | 自定义智能体名称 |
| `thread_id` | str | — | 线程 ID（由 Runtime 注入） |

---

> 本文档：`make_lead_agent()` 是每次请求调用的工厂函数，五步组装产出编译图。模型名称按请求参数 > 智能体配置 > 全局默认三级解析，不支持 thinking 的模型自动降级。运行时参数通过 `config.configurable` 传递
